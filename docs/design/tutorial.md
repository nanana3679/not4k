# not4k — 튜토리얼 설계

이 문서는 노트 연주법과 판정 방식을 단계적으로 소개하는 튜토리얼의 설계를 정의한다.

> **키 바인딩 튜토리얼과의 관계**: `keybinding.md`의 튜토리얼 흐름은 물리적 키 배치의 발견(4키 → 6키 → 8키)에 집중한다. 이 문서의 튜토리얼은 노트 메커니즘과 판정 규칙의 학습에 집중한다. 두 트랙은 병행 또는 순차적으로 진행될 수 있으며, 구체적인 결합 방식은 TBD.

---

## 세분화의 이유

not4k의 입력과 판정에는 직관적이지 않은 부분이 존재한다.

- 롱노트가 이어진 경우(`o-o-`)에는 릴리즈 판정이 존재하지 않고 유지 판정으로 대체된다.
- 릴리즈 판정에서 Good 이상은 Perfect로 상향 처리된다.
- 트릴 노트에서 교대 실패 시 Good으로 고정된다.
- 싱글→더블 전환(`o-o=`)에서 전환 시점에 릴리즈가 발생하지 않는다.
- 더블→싱글 전환(`o=o-`)에서는 릴리즈가 발생한다.
- 분할 릴리즈(`o=-`)는 더블 롱노트에서만 가능하다.

이러한 규칙을 한 번에 제시하면 유저가 압도된다. **최초 튜토리얼에서는 플레이만 가능하도록 노트의 연주 방법만 소개하고, 후속 튜토리얼에서 복합 패턴을 소개하며 판정 방식을 알려줘야 한다.**

---

## 튜토리얼 구조

### Phase 1: 연주 방법 — "이렇게 치면 된다"

노트 타입별 기본 입력 방법만 소개한다. 판정의 세부 규칙은 언급하지 않는다.

| 항목 | 소개하는 것 | 소개하지 않는 것 |
|------|------------|-----------------|
| 싱글 노트 | 키를 누른다 | 판정 윈도우 수치 |
| 롱 노트 | 키를 누르고 있다가 뗀다 | 넘어가기(키 릴레이), 유예 시간, 릴리즈 상향 |
| 트릴 노트 | 트릴 구간에서 키를 번갈아 누른다 | 교대 실패 시 Good 고정 |
| 더블 노트 | 다른 키 2개를 동시에 누른다 | 독립 판정 × 2, 배드말림 해제 |

목표: 유저가 **모든 노트 타입을 처리할 수 있는 상태**가 된다. "왜 이 판정이 나왔는지"는 아직 알 필요 없다.

### Phase 2: 복합 패턴과 판정 — "왜 이렇게 되는가"

Phase 1의 연주 능력을 전제로, 복합 패턴에서 발생하는 판정 규칙을 소개한다.

- 롱노트 연결 시 릴리즈 판정 vs 유지 판정의 차이
- 릴리즈 판정의 Good 이상 → Perfect 상향
- 트릴 교대 실패 시 Good 고정 (TGood)
- 전환 릴리즈: 키 수 변화에 따른 릴리즈 발생 규칙
- 분할 릴리즈(`o=-`)의 판정 횟수

목표: 유저가 **자신의 판정 결과를 이해하고 개선 방향을 파악할 수 있는 상태**가 된다.

### Phase 3: 넘어가기 — "홀드 중에 키를 바꿀 수 있다"

넘어가기(키 릴레이)는 추가 키를 사용해야 하는 동작이므로 반드시 튜토리얼이 필요하다. 롱노트 진행 중에 다른 키로 홀드를 이어받는 것은 기존 리듬게임에 없는 개념이며, 유저가 스스로 발견하기 어렵다. 고난이도에서만 사용되므로 판정 규칙 학습 이후에 배치한다.

- 롱노트를 A로 유지하다가 B로 넘어가도 끊기지 않는다는 것을 체험시킨다.
- 유예 시간(grace period)의 존재를 암시한다: A를 떼고 B를 약간 늦게 눌러도 유지된다.
- 넘어가기가 롱+싱글 공존 피스의 전제 조건임을 소개한다.

목표: 유저가 **롱노트 중 키 전환이 가능하다는 것을 체득한 상태**가 된다.
