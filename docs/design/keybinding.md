# not4k — 키 바인딩 시스템

---

## 손배치 철학: 약중검엄

### 소중약검(4키 리듬게임의 표준)의 문제

기존 4키 PC 리듬게임에서 유저는 소지-약지-중지-검지를 일렬로 배치한다. 그러나 이 배치에는 인체공학적 문제가 있다.

- **소지의 부자연스러움**: 소지를 다른 손가락과 일렬로 배치하려면 손을 비틀어야 하며, 이 자세를 장시간 유지하면 손목에 부담이 가중된다.
- **소지의 약한 힘**: 소지는 다른 손가락에 비해 근력과 독립성이 현저히 떨어진다. 고밀도 차트에서 소지 레인이 병목이 되는 경우가 빈번하다.

### 약중검엄 배치

유저가 키보드에 손을 올려두었을 때, 약지-중지-검지는 자연스럽게 일렬로 배치되고, 엄지는 하단에 위치한다. 이것이 인체의 자연스러운 손 형태이며, not4k는 이 배치를 전제로 설계한다.

```
   약  중  검          ← 상단 행 (윗줄 키)
   약  중  검          ← 홈 행 (가운데줄 키)
   약  중  검          ← 하단 행 (아랫줄 키)
            엄         ← 최하단 (아랫줄 키 or 스페이스바 근처)
```

이 배치의 장점:

- **엄지 활용**: 소지 대신 엄지가 4번째 손가락 역할을 한다. 엄지는 소지보다 힘이 강하고, 키보드 하단에 자연스럽게 위치하므로 별도의 자세 교정이 필요 없다.
- **선례**: DJMAX, 식스타 게이트, 사운드 볼텍스(약지 제외) 등에서 이미 사용되어온 검증된 배치다.
- **트릴 확장의 자연스러움**: 약지 위치에 레인 키를 추가하면, 약중 교대 트릴이 자연스럽게 가능해진다(아래 상세 설명).

## 키보드 레이아웃

QWERTY 키보드 기준으로, not4k가 사용하는 키 영역은 다음과 같다.

```
   Q  W  E        7  8  9
   A  S  D        4  5  6
   Z  X  C        1  2  3
```

왼손은 QWE / ASD / ZXC 영역, 오른손은 789 / 456 / 123 영역(넘버패드)을 사용한다.

각 행은 손가락의 상하 이동에 대응한다:

- **상단 행** (Q W E / 7 8 9): 약지-중지-검지가 위로 뻗는 위치
- **홈 행** (A S D / 4 5 6): 약지-중지-검지의 기본 위치
- **하단 행** (Z X C / 1 2 3): 약지-중지-검지가 아래로 내려가는 위치. 엄지도 이 영역에 접근 가능

### 넘버패드의 인체공학적 근거

오른손 넘버패드 배치에서 3번 레인은 7(검지)과 1(엄지)을 사용한다. 7과 1 사이에는 3행(7→4→1)의 갭이 존재하는데, 이는 검지와 엄지의 손가락 길이 차이를 반영한 것이다. 검지가 상단 행(7)에 위치할 때 엄지는 자연스럽게 하단 행(1)에 닿으며, 중간 행(4)을 건너뛰는 것이 오히려 인체공학적으로 자연스럽다.

### 텐키리스 대안 배치 (TKL / 65% / 75%)

넘버패드가 없는 키보드 사용자를 위해, 오른손에 메인 키보드 영역의 3×3 블록을 대안으로 제공한다.

```
   Q  W  E        P  [  ]
   A  S  D        L  ;  '
   Z  X  C        ,  .  /
```

넘버패드 키와의 대응 관계:

| 넘버패드 | TKL 대안 | 행      |
| -------- | -------- | ------- |
| 7 8 9    | P [ ]    | 상단 행 |
| 4 5 6    | L ; '    | 홈 행   |
| 1 2 3    | , . /    | 하단 행 |

이 배치의 근거:

- **3×3 그리드 보존**: P[]/L;'/,./ 는 물리적으로 3행 × 3열 블록을 형성하므로, 넘버패드와 동일한 손가락 배치(검지-중지-약지)와 상하 이동 구조가 그대로 유지된다.
- **왼손과의 대칭성**: 왼손(QWE/ASD/ZXC)이 키보드 왼쪽 끝의 3×3 블록을 사용하듯, 오른손은 키보드 오른쪽 끝의 3×3 블록을 사용한다. 양손이 키보드의 양쪽 끝에 대칭적으로 위치하게 된다.
- **모든 폼팩터 호환**: 60%, 65%, 75%, TKL 등 넘버패드가 없는 레이아웃에서도 이 키들은 항상 존재한다.
- **손가락 역할 동일**: 각 열의 손가락 배정(검지=좌측열, 중지=중앙열, 약지=우측열)이 넘버패드 배치와 동일하므로, 두 배치 간 전환 시 재학습이 필요 없다.

게임 설정에서 "오른손 배치"를 넘버패드 또는 TKL 중 선택할 수 있으며, 기본값은 TKL 배치로 한다. 대다수의 게이밍 키보드가 텐키리스 계열이므로, 별도 설정 없이 바로 플레이할 수 있어야 한다.

## 난이도 등급별 키 배치

### 입문 (Lv.1~3): 4키 주로 활용, 드물게 엄지

16키 프리셋(레인당 4키)을 기본으로 설정하되, 입문 단계에서는 중지와 검지 4키를 주로 활용한다.

- **레인 1, 4**: 중지 1키만 주로 사용.
- **레인 2, 3**: 검지 1키를 주로 사용. 트릴·더블 노트가 극소량 등장할 때만 엄지를 드물게 사용.

근거: 기존 4K 유저가 거의 그대로 플레이할 수 있는 수준을 유지하면서, "이 게임은 다르다"는 인식만 심어준다.

| 레인 | 왼손 키 (손가락)    | 오른손 키 — 넘버패드 (손가락) | 오른손 키 — TKL (손가락) | 입문 활용도       |
| ---- | ------------------- | ----------------------------- | ------------------------ | ----------------- |
| 1    | Q (약지) + W (중지) |                               |                          | 주로 W만          |
| 2    | E (검지) + C (엄지) |                               |                          | 주로 E만, 드물게 C |
| 3    |                     | 7 (검지) + 1 (엄지)           | P (검지) + , (엄지)      | 주로 7/P만, 드물게 1/, |
| 4    |                     | 8 (중지) + 9 (약지)           | [ (중지) + ] (약지)      | 주로 8/[만        |

### 초급 (Lv.4~6): 엄지 비중 상승 (6키)

트릴 노트가 빈번해지면서 내측 레인(2, 3번)에서 엄지 사용이 본격화된다. 외측 레인(1, 4번)은 아직 주로 1키만 사용.

### 중급 (Lv.7~10): 8키 입문 — 약지 추가

외측 레인(1, 4번)에도 트릴 노트가 등장하면서 약지를 추가한다. 8키 전체를 사용하기 시작하는 단계.

| 레인 | 왼손 키 (손가락)    | 오른손 키 — 넘버패드 (손가락) | 오른손 키 — TKL (손가락) |
| ---- | ------------------- | ----------------------------- | ------------------------ |
| 1    | Q (약지) + W (중지) |                               |                          |
| 2    | E (검지) + C (엄지) |                               |                          |
| 3    |                     | 7 (검지) + 1 (엄지)           | P (검지) + , (엄지)      |
| 4    |                     | 8 (중지) + 9 (약지)           | [ (중지) + ] (약지)      |

각 레인의 트릴 처리:

- **1번 레인**: Q-W 교대 (약지-중지)
- **2번 레인**: E-C 교대 (검지-엄지)
- **3번 레인**: 7-1 교대 (검지-엄지) / TKL: P-, 교대 (검지-엄지)
- **4번 레인**: 8-9 교대 (중지-약지) / TKL: [-] 교대 (중지-약지)

### 기본 프리셋: 16키 (레인당 4키)

초기 제공 프리셋은 레인당 4키를 바인딩한다. 외측 레인(1, 4)은 한 손의 중지 3행 + 약지 1키, 내측 레인(2, 3)은 검지 + 엄지 기반에 반대 손의 브릿지 키 1개를 추가한 구성이다.

| 레인 | 왼손 키                        | 오른손 키 — 넘버패드          | 오른손 키 — TKL              |
| ---- | ------------------------------ | ----------------------------- | ---------------------------- |
| 1    | Q (약 상단) + W (중 상단) + S (중 홈행) + X (중 하단) | —                             | —                            |
| 2    | E (검 상단) + D (검 홈행) + C (엄지)                  | PgDn                          | O                            |
| 3    | R                              | 7 (검 상단) + 4 (검 홈행) + 1 (엄지) | P (검 상단) + L (검 홈행) + , (엄지) |
| 4    | —                              | 8 (중 상단) + 9 (약 상단) + 5 (중 홈행) + 2 (중 하단) | [ (중 상단) + ] (약 상단) + ; (중 홈행) + . (중 하단) |

**내측 레인의 크로스핸드 키**: 2번 레인의 PgDn/O와 3번 레인의 R은 반대 손에 위치한다. 평상시에는 같은 손의 3키(검지 + 엄지)로 충분하나, 고밀도 구간에서 반대 손의 여유 손가락이 보조 입력으로 참여할 수 있는 선택지를 제공한다.

4키 바인딩의 이점:

- **중지 3행 활용**: 외측 레인에서 중지가 상단·홈·하단 3행을 모두 사용하여 트릴 선택지가 넓어진다
- **3키 트릴**: 24비트 등 3박자 기반 트릴에서 `A → B → C → A → B → C` 교대가 자연스럽다
- **4키 트릴**: 32비트 고속 트릴에서 `A → B → C → D → A → B → C → D` 교대로 각 손가락의 회복 시간을 극대화한다
- **더블 홀드 + 트릴**: 4키 중 2키로 더블 홀드를 유지하면서 나머지 2키로 트릴 처리가 가능하다
- **크로스핸드 유연성**: 내측 레인의 고밀도 구간에서 반대 손이 보조할 수 있다

## 트릴 키 배치의 인체공학적 근거

### 외측 레인 (1번, 4번): 약지-중지 트릴

약지와 중지는 키보드 같은 행의 인접 키에 위치하므로, 교대 입력이 자연스럽다. 두 손가락의 힘줄이 비교적 독립적이어서 빠른 교대에도 간섭이 적다.

### 내측 레인 (2번, 3번): 검지-엄지 트릴

검지는 홈 행에, 엄지는 하단 행에 자연스럽게 위치한다. 검지와 엄지는 해부학적으로 완전히 독립적인 근육으로 제어되므로, 트릴 속도의 상한이 높다. DJMAX 컨트롤러에서 엄지를 적극 활용하는 것과 동일한 원리다.

### 고급 테크닉: 엄지 눕힘

IIDX의 엄검엄/뼈검엄 파지법처럼 엄지를 눕혀 인접 키 2개를 동시에 처리하는 테크닉.

- 검중-약엄 트릴보다 검중-엄(눕힘) 트릴이 훨씬 수월함
- 16bit `12 12 12 12` 같은 배치에서 엄지를 눕혀 1,2번 레인을 한 손가락으로 처리 가능
- 최상급 난이도에서 유용한 고급 입력 방식

### 소지를 배제하는 이유

- 소지와 약지는 힘줄이 연결되어 있어 독립적인 움직임이 어렵다. (IIDX 4756 문제의 근본 원인과 동일)
- 소지를 일렬 배치에 포함시키면 손을 비틀어야 하며, 이 자세의 장시간 유지는 손목 부담을 가중시킨다.
- not4k는 소지 대신 엄지를 활용함으로써, 기존 리듬게임의 소지 관련 인체공학적 문제를 회피한다.

## 튜토리얼 흐름

not4k의 튜토리얼은 유저에게 16키 프리셋(레인당 4키)을 기본으로 설정하게 한 뒤, 4키 → 6키(엄지) → 8키(약지) → 16키로 활용도를 자연스럽게 확장하는 것을 목표로 한다.

### Step 1: 기본 4키 (한 행씩 체험)

유저에게 세 가지 행을 순서대로 체험시킨다. 각 행에서 4비트 간격의 단순한 차트를 플레이한다.

```
Step 1-1: 상단 행  → 왼(W E) 오(7 8 또는 P [)로 1 2 3 4 노트 처리
Step 1-2: 홈 행    → 왼(S D) 오(4 5 또는 L ;)로 1 2 3 4 노트 처리
Step 1-3: 하단 행  → 왼(X C) 오(1 2 또는 , .)로 1 2 3 4 노트 처리
```

목적: 키보드의 세 행 모두에서 중지-검지 배치로 4키를 처리할 수 있다는 것을 체험시킨다. 이 단계에서는 아직 트릴 노트가 등장하지 않는다.

### Step 2: 트릴 노트 도입 — 약지의 발견

1번 레인에 연속 노트가 등장하는 차트를 제시한다.

4bit`1 1 1 1`

먼저 W 키 연타로 처리하게 한 뒤, "키를 하나 더 추가해보세요"라는 안내를 제공한다.

유저가 1번 레인에 W(중지)만 바인딩한 상태에서, Q(약지)를 추가하도록 유도한다.

```
연타: W W W W (중지만 사용, 손가락 피로)
트릴: Q W Q W (약지-중지 교대, 자연스러움)
```

여기서 핵심 체감이 발생한다: **W-S-W-S(중지로 상하 이동)보다 Q-W-Q-W(약중 교대)가 훨씬 자연스럽다.** 상하 이동은 손 전체가 움직여야 하지만, 약중 교대는 손가락만 움직이면 된다.

### Step 3: 엄지의 발견

Step 2를 학습한 유저에게 2번 레인의 트릴을 도입한다.

4bit`2 2 2 2`

2번 레인에 E(검지)만 바인딩된 상태에서, "키를 하나 더 추가해보세요"라는 안내를 제공한다. 유저는 자연스럽게 하단 행의 C에 엄지가 닿는다는 것을 발견한다.

```
트릴: E C E C (검지-엄지 교대)
```

이 시점에서 유저의 왼손 배치가 자연스럽게 완성된다:

```
왼손: Q(약) W(중) E(검) C(엄)
      1번레인(QW)  2번레인(EC)
```

### Step 4: 양손 완성

오른손도 동일한 과정을 거친다.

넘버패드 배치:

```
3번 레인 트릴: 7-1 교대 (검지-엄지)
4번 레인 트릴: 8-9 교대 (중지-약지)
```

TKL 배치:

```
3번 레인 트릴: P-, 교대 (검지-엄지)
4번 레인 트릴: [-] 교대 (중지-약지)
```

최종 배치:

넘버패드:

```
왼손: Q(약) W(중) E(검) C(엄)  →  1번 레인(QW), 2번 레인(EC)
오른손: 7(검) 1(엄) 8(중) 9(약)  →  3번 레인(71), 4번 레인(89)
```

TKL:

```
왼손: Q(약) W(중) E(검) C(엄)  →  1번 레인(QW), 2번 레인(EC)
오른손: P(검) ,(엄) [(중) ](약)  →  3번 레인(P,), 4번 레인([])
```

## 핵심 규칙

### 1. 레인당 복수 키 바인딩 필수

- 유저는 각 레인에 **2개 이상**의 키를 바인딩**해야** 한다 (최소 요구사항).
- **최소**: 레인당 2키 (총 8키). 트릴과 더블 노트 처리를 위한 최소 조건.
- **기본 프리셋**: 레인당 4키 (총 16키). 초기 제공 프리셋에 적용. 약중검엄 배치에서 상단 행 + 홈 행을 완전 활용하고, 내측 레인은 엄지 인접 키를 추가한 구성.
- **확장**: 레인당 4키 이상 바인딩도 **허용한다.** 시스템은 바인딩 수의 상한을 두지 않는다.

### 2. 바인딩된 키는 모두 동등

- 같은 레인에 바인딩된 키 A, B는 싱글 노트에서 완전히 동등하게 작동한다.
- "A가 주키, B가 보조키"라는 구분은 없다.
- 트릴 노트에서만 "직전과 다른 키"라는 조건이 붙는다.

### 3. 키 충돌 불가

- 하나의 물리 키는 하나의 레인에만 바인딩된다.
- 같은 키를 두 개 이상의 레인에 바인딩할 수 없다.

## UI 설계 요구사항 (미정)

### 키 바인딩 설정 화면

- 레인별로 바인딩된 키를 시각적으로 보여주는 인터페이스
- 키보드 레이아웃 위에 현재 바인딩을 오버레이하여 시각적으로 확인 가능
- 현재 바인딩으로 플레이 가능한 난이도 등급을 표시

### 곡 선택 화면

- 각 곡/차트의 Lv.를 표시
- 현재 바인딩으로 처리할 수 없는 노트 타입이 포함된 경우 안내 메시지 표시

### 플레이 중 표시

- 현재 바인딩된 키 구성을 플레이 화면 하단에 작게 표시하는 옵션
- 트릴 노트 구간에서 교대 가이드를 표시하는 옵션 (초급자용, 끌 수 있음)
