# not4k — 이 게임은 왜 존재하는가

## 한 줄 정의

> **not4k는 4레인이라는 단순한 시각 구조 위에 "레인당 다중키 바인딩"을 도입함으로써, 기존 PC 4키 리듬게임의 세 가지 구조적 한계 — 연타의 물리적 고통, 키 수 간 난이도 단절, 고정 손배치가 만드는 불가능 패턴 — 를 동시에 해결하는 가변 손배치 리듬게임이다.**

---

## 1. 기존 리듬게임이 가진 구조적 문제

아래 세 가지 문제는 특정 게임 하나의 결함이 아니라, PC 노트낙하형 4키 리듬게임이라는 장르가 공유하는 구조적 한계다. 각 문제의 근거는 `../research/`의 역기획 보고서에서 확인할 수 있다.

### 문제 A: 연타가 패턴의 다양성을 죽인다

4키 고정배치 리듬게임에서 난이도를 올리는 가장 직접적인 수단은 연타다. 그런데 연타 자체가 이미 높은 처리 난이도를 가지기 때문에, 연타가 등장하는 구간에서는 다른 복잡한 입력을 함께 요구하기 어렵다. 결과적으로 고난이도에서도 연타 구간의 패턴은 단순해질 수밖에 없다.

동시에 연타는 손목에 반복적인 충격을 가하여 건초염, 손목 터널 증후군 등의 원인이 된다.

**근거**:

- IIDX: 손목 스크래치(Tekubi)의 부상 위험, 건초염 사례 (`../research/beatmania.md` 6.2절)
- DJMAX 5B: 손가락 경직(Locking up) 현상 보고 (`../research/djmax.md` Part 2-B)
- 사운드 볼텍스: 연전 시 손목 부담, "건강 제일" 커뮤니티 문화 (`../research/soundvoltex.md`)
- 츄니즘: 슬라이더 연타 시 물리적 반발력 부재로 인한 피로 가중 (`../research/chunithm.md` Part 2-B)

### 문제 B: 키 수가 다르면 난이도가 호환되지 않는다

DJMAX에서 4B를 수백 시간 한 유저가 6B를 시작하면 사실상 처음부터 다시 적응해야 한다. 키 수가 다른 모드는 사실상 다른 게임이며, 하나의 통합된 성장 경로를 제공하지 못한다.

**근거**:

- DJMAX 5B: "4B→5B 전환이 가장 고통스러운 구간"이라는 분석, "6B로 먼저 건너뛴 후 5B로 내려오는 것이 효율적 경로"라는 커뮤니티 합의 (`../research/djmax.md` Part 2-D)
- IIDX: 4키 리듬게임 유저가 7+1키 구조에 적응 실패하는 사례 (`../research/beatmania.md` 8.1절)
- 츄니즘: 4K 스타일에서 8K 미세노트로의 전환 시 "패닉" 보고 (`../research/chunithm.md` Part 2-D)

### 문제 C: 고정 손배치가 물리적으로 불가능한 패턴을 만든다

IIDX의 4756 문제, PC 리듬게임의 거미줄/데님 패턴은 인간의 해부학적 한계와 정면으로 충돌한다. 유저들은 이를 회피하기 위해 랜덤 옵션으로 "황배"를 노리는데, 이것은 실력이 아닌 운에 클리어 여부를 맡기는 것이다.

**근거**:

- IIDX: 4756 데님 패턴의 해부학적 한계, 랜덤 옵션의 필수화 (`../research/beatmania.md` 5장, 6.3절)
- DJMAX 5B: "홀수 비대칭 계단 패턴"의 인지 부하 (`../research/djmax.md` Part 1-B-3)
- 사운드 볼텍스: BT+FX 복합 패턴에서의 손가락 독립성 한계 (`../research/soundvoltex.md`)

## 2. 아케이드 게임이 보여준 해법의 단서

PC 리듬게임과 달리, 아케이드 리듬게임들은 물리적 인터페이스의 특성 덕분에 위 문제들을 부분적으로 해결하고 있다.

| 게임              | 해법                                                                                                          | 한계                                                                                     |
| ----------------- | ------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| **사운드 볼텍스** | BT 버튼이 가깝게 배치되어 손배치가 자유로움. 한 레인의 연타를 양손으로 분할 처리 가능                         | 아케이드 전용. PC(코나스테)에서는 전용 컨트롤러가 필요하며 가격이 높음                   |
| **DJMAX 5B**      | 중앙 레인(3번)에 좌우 양손의 키가 공유됨. 중앙 연타를 트릴로 전환 가능                                        | 가변 손배치가 중앙 레인에서만 작동. 저난이도에서는 활용할 기회가 없어 학습 곡선이 급격함 |
| **츄니즘**        | 물리적 칸막이가 없는 슬라이더로 타격 지점을 자유롭게 변경 가능. 연타를 양손 트릴로 전환하거나 "비비기"로 처리 | 아케이드 전용. 슬라이더라는 하드웨어에 의존하며 PC로 이식 불가                           |
| **maimai**        | 8버튼 원형 배치로 손배치가 매 순간 변화. 교차 패턴에서 최적 동선을 유저가 실시간 계산                         | 아케이드 전용. 전신 운동에 가까운 조작 체계로 PC 환경과 완전히 이질적                    |

이 게임들의 공통점은 **유저가 특정 입력 지점에 고정되지 않고, 상황에 따라 어떤 손/손가락으로 처리할지 선택할 수 있다**는 것이다. not4k는 이 원리를 PC 키보드 환경에서 "레인당 다중키 바인딩"이라는 소프트웨어적 메커니즘으로 구현한다.

## 3. not4k의 해법

### 핵심 원칙: 4레인 + 레인당 다중키 바인딩

- 시각적으로는 항상 4개의 레인만 존재한다.
- 유저는 각 레인에 복수의 키를 바인딩할 수 있다. (예: 1번 레인에 D와 F)
- 같은 레인의 연속 노트를 하나의 키로 연타할 수도, 다른 키와 교대하여 트릴로 처리할 수도 있다.
- 어떤 키를 사용할지는 유저의 선택이다.

### 문제 A → 연타가 트릴이 된다

16bit`2 2 2 2`가 트릴로 처리 가능해지면, 이 패턴이 진행되는 동안 다른 레인에 복잡한 입력을 함께 배치할 수 있다. 연타가 패턴의 다양성을 제한하던 구조적 제약이 사라지고, 손목에 가해지는 반복 충격도 분산된다.

### 문제 B → 레인은 항상 4개, 키 수만 유저가 선택

레인이 항상 4개이므로, 4키 유저와 8키 유저가 같은 노트를 보고 같은 난이도 체계를 공유한다. DJMAX에서 4B→6B→8B로 갈 때마다 단절되던 성장 곡선이 하나의 연속된 경사로 연결된다. 차트의 복잡도가 올라가면 자연스럽게 더 많은 키를 바인딩하게 되지만, 레인 구조 자체는 변하지 않는다.

### 문제 C → 고통스러운 움직임이 강제되지 않는다

어떤 키 조합으로 입력할지는 유저의 선택이다. 4756 같은 해부학적 한계에 부딪히는 손가락 조합이 시스템에 의해 강제되지 않으므로, 랜덤으로 황배를 노리는 행위 자체가 불필요해진다.

## 4. 기존 게임과의 차별화

| 비교 대상         | not4k와의 관계                                                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| **osu!mania 4K**  | 싱글 노트만으로 구성된 패턴에서는 완벽히 호환. 추가 학습 없이 즉시 플레이 가능. 트릴·더블 등이 등장하면서 가변 손배치가 시작됨 |
| **DJMAX 5B**      | 중앙 레인 공유를 모든 레인으로 확장. 트릴 노트로 입문 등급(Lv.1~3)에서도 가변 손배치 학습 가능                                 |
| **사운드 볼텍스** | 아케이드의 가변 손배치를 PC 키보드 환경으로 이식. 노브 없이 키보드만으로 유사한 자유도 확보                                    |
| **츄니즘**        | 슬라이더의 "면적 입력"이 제공하는 자유도를 키보드의 "다중키 바인딩"으로 치환. 한 레인 내 홀드 중 탭(PLv.7) 차용              |
| **IIDX**          | 4756 문제와 랜덤 의존성을 유저 선택에 의한 키 배분으로 근본적으로 해결                                                         |

## 5. 인식 난이도라는 트레이드오프

가변 손배치는 입력 난이도를 낮추는 대신 인식 난이도를 올린다. "이 노트를 어떤 손/키로 처리할 것인가"를 실시간으로 판단해야 하기 때문이다. DJMAX 5B 초보자가 중앙 레인 판단에서 0.1초 지연으로 미스가 나는 것이 이 트레이드오프의 전형적인 사례다.

not4k는 이를 두 가지 장치로 관리한다.

1. **트릴 노트**: "이 구간에서는 반드시 키를 교대로 사용해야 한다"는 것을 시각적으로 명시하여, 유저가 실시간으로 판단해야 하는 인지 부하를 줄인다.

2. **난이도 계단**: 입문 등급(Lv.1~3)에서는 기존 4K와 동일하게 플레이하고, 트릴 노트를 통해 점진적으로 가변 손배치를 학습한 뒤, 중급 이상(Lv.7~)에서 비로소 완전한 가변 손배치를 활용하는 구조. 인식 난이도의 급격한 상승을 방지한다.
