# not4k — 용어 사전

이 문서는 not4k 설계 문서 전체에서 사용하는 용어를 정의한다. 같은 단어가 다른 맥락에서 다른 의미로 사용되는 혼동을 방지하기 위해 작성되었다.

---

## 게임 구조

### 레인 (Lane)

화면에 표시되는 노트 낙하 경로. not4k에는 항상 4개의 레인(L1~L4)이 존재한다. 레인의 수는 변하지 않으며, 유저가 바인딩하는 키의 수만 변한다.

### 레인 분할 (Lane Split)

유저가 양손으로 4개의 레인을 어떻게 나누어 담당하는지를 가리킨다. 기존 4키 게임은 2/2 고정이지만, not4k에서는 트릴 노트의 존재로 인해 1/3, 3/1, 0.5/3/0.5 등 가변적인 분할이 가능해진다.

### 틱 (Tick)

채보에서 시간의 최소 단위. 비트 분할(4비트, 8비트, 16비트)에 따라 틱 간격이 달라진다. 피스 표기법에서 한 행이 한 틱에 대응한다.

### 비트 분할 (Bit Division)

한 박자를 몇 등분하는지를 나타낸다. 4비트는 1박 1노트, 8비트는 1박 2노트, 16비트는 1박 4노트. BPM과 결합하여 초당 노트 수(노트 밀도)를 결정한다.

---

## 입력 체계

### 다중키 바인딩 (Multi-Key Binding)

하나의 레인에 2개 이상의 물리 키를 할당하는 not4k의 핵심 메커니즘. 이것이 트릴, 더블, 홀드 중 탭 등의 피스를 가능하게 한다.

### 가변 손배치 (Variable Hand Position)

유저가 채보의 흐름에 따라 손가락 배치를 실시간으로 바꾸는 플레이 방식. not4k의 정체성을 정의하는 핵심 개념이다. 키 수 자체는 고정이지만, 어떤 손가락이 어떤 역할을 하는지가 차트에 따라 달라진다.

### 약중검엄

not4k가 전제하는 손가락 배치 — 약지, 중지, 검지, 엄지의 4손가락을 사용하고 소지를 배제한다. 인체공학적으로 자연스러운 자세를 유지하면서 레인당 2키 바인딩을 구현한다.

### 키 릴레이 (Hold Transfer)

롱노트 진행 중에 같은 레인의 다른 키로 홀드를 이어받는 메커니즘. 유예 시간(Grace Period) 내에 전환하면 롱노트가 끊기지 않는다. 홀드 중 탭의 기반이 되는 메커니즘이다.

### 홀드탭 (Hold-Tap)

홀드 중 탭(`-o`) 배치를 처리하는 입력 방식. 한 키(A)로 롱노트 홀드를 유지한 채 다른 키(B)로 싱글 노트를 탭한다. 두 키가 동시에 활성 상태가 된다. 가장 직관적인 처리 방식.

### 릴리즈 교차 (Release-Tap)

한 레인에서 롱노트가 끝날 때, 유지하던 키(A)를 떼면서 같은 레인의 다른 키(B)를 누르는 행위. 두 키를 번갈아 입력하는 키 교대 동작이다. 이 입력을 요구하는 배치를 **릴리즈 교차 피스**(PLv.4)라 한다.

홀드 중 탭(`-o`) 배치에서 홀드탭의 대안 처리 방식이기도 하다. A(hold) → A(release) + B(hold) → B(release) + A(hold)으로 키를 번갈아 사용한다. 키 릴레이의 유예 시간 내에 전환하므로 롱노트가 끊기지 않는다.

## 노트 타입

> **원칙: 모든 노트 타입은 Lv.1부터 등장할 수 있다.** 노트 타입 자체에는 최소 레벨 제한이 없다. 난이도 제한은 노트 타입이 아니라 피스(노트 배치의 상호작용)에 적용된다.

### 싱글 노트 (Single Note)

레인에 바인딩된 키 중 아무 키 하나를 누르는 기본 노트. 표기: `o`

### 롱 노트 (Long Note)

키를 누르고 유지하다가 끝 지점에서 떼는 노트. 헤드와 바디로 구성된다. 헤드는 싱글 또는 더블 노트이며, 바디는 `-`(싱글) 또는 `=`(더블)로 표기한다. 릴리즈는 바디 심볼이 끝나는 시점에서 자동으로 발생한다.

- **헤드 (Head)**: 롱노트의 시작 지점. 싱글 또는 더블 노트로 구성되며, 해당 노트 타입의 판정 규칙이 적용된다.
- **바디 (Body)**: 헤드 이후 키를 유지하는 구간. Good 윈도우 내에 입력이 시작되어야 하며, 릴리즈 시점에서 판정이 발생한다.
- **판정 횟수**: 싱글 롱노트는 최대 2회 (헤드 1회 + 릴리즈 1회). 더블 롱노트는 최대 4회 (헤드 2회 + 릴리즈 2회).
- **헤드-바디 타입 혼합**: 싱글 헤드 + 더블 바디, 더블 헤드 + 싱글 바디 등의 조합이 가능하다.

### 트릴 노트 (Trill Note)

트릴 구간 안에서 직전 입력과 다른 키를 눌러야 하는 노트. 2개 이상의 키 바인딩이 필요하다. 표기: `t`

### 트릴 구간 (Trill Zone)

트릴 노트가 등장할 수 있는 시각적 영역. 트릴 구간 자체는 입력을 요구하지 않으며, 예고·시각적 구분·긴장도 유지의 역할을 한다. 구간 내 빈 틱에서의 입력은 구간 밖의 빈 틱과 동일하게 동작한다 — 트릴 구간이 있든 없든 결과가 같다. 표기: `{` (시작) / `}` (종료) / `~` (구간 내 빈 틱)

### 더블 노트 (Double Note)

같은 레인에 바인딩된 서로 다른 키 2개를 동시에 누르는 노트. 같은 키를 두 번 누르는 더블클릭은 더블 노트로 인정되지 않는다. 키 바인딩이 3개 이상인 레인에서는 바인딩된 키 중 서로 다른 아무 2개를 사용할 수 있다. 임팩트 강조와 플레이 다양성 확장의 역할을 수행한다. 표기: `D`

## 피스

### 피스 (piece)

복수의 노트가 배치를 통해 상호작용하여 난이도를 발생시키는 단위. 개별 노트 타입과 달리, 피스는 각각 고유한 PLv.(피스 레벨)를 가진다. `piece-definition.md`의 피스 카탈로그에서 `PP-001`, `PP-003` 등의 ID로 관리한다.

**노트 타입과 피스의 관계**: 노트 타입은 개별 요소(싱글, 더블, 트릴, 롱)이며 Lv.1부터 등장한다. 피스는 이 요소들의 배치가 상호작용하여 난이도를 만들어내는 것이다. 예를 들어, 롱노트와 싱글 노트는 각각 Lv.1부터 등장하지만, 롱노트가 끝날 때 키 교대를 요구하는 배치(릴리즈 교차)는 PLv.4의 피스이다.

> 구 용어: 패턴 조각 (Pattern Piece).

**용례**:

- "Lv.8 채보의 사비 구간에 PLv.7 피스 PP-003이 배치되어 있다"
- "트릴 앵커와 레인 내 분리를 결합한 고난이도 피스"

### 대표 피스

아래는 문서 전반에서 자주 참조되는 피스의 간략한 정의이다. 정식 피스 정의(PP-000 체계)는 `piece-definition.md`에서 관리한다.

- **릴리즈 교차 (Release-Tap)**: 롱노트가 끝날 때 같은 레인에서 키 교대를 요구하는 배치. 구성 노트 타입: 싱글 + 롱. PLv.4. `-` 다음 틱에 `o`가 오는 것으로 표현된다.
- **홀드 중 탭 (Tap-in-Hold)**: 한 키로 롱노트를 유지하면서 같은 레인의 다른 키로 싱글 노트를 처리하는 배치. 구성 노트 타입: 싱글 + 롱. PLv.7. 표기 예시: `-o`

### 피스 메타데이터

피스를 문서화할 때 함께 기록하는 속성들.

| 필드               | 설명                                                     |
| ------------------ | -------------------------------------------------------- |
| `id`               | 피스 고유 식별자 (예: `PP-001`)                          |
| `name`             | 피스 명칭 (예: "릴리즈 교차")                            |
| `description`      | 어떤 노트 간 상호작용이 난이도를 발생시키는지 서술       |
| `note_types`       | 이 피스를 구성하는 노트 타입 목록                        |
| `plv`              | 피스 레벨(PLv.). 이 피스가 본격 활용되는 최소 차트 레벨  |
| `notation_example` | 피스 표기법 스니펫                                       |
| `judgment_notes`   | 일반 규칙과 다른 판정 특이사항. 없으면 생략              |
| `prerequisite`     | 선행 피스. 이 피스 수행 전에 유저가 숙련되어야 하는 피스 |
| `lane_key_req`     | 최소 레인 수, 최소 키 바인딩 수 등의 요구사항            |
| `remarks`          | 설계 의도, 참고 게임 등의 비고                           |

## 채보 설계

### 채보 (Chart)

한 곡에 대한 노트 배치 데이터 전체. 곡 하나에 난이도별로 복수의 채보가 존재할 수 있다. "이 곡의 Lv.8 채보"처럼 쓴다. "차트"는 채보의 동의어이며, "차트 디자인", "차트 레벨" 등 복합어에서 주로 사용한다.

### 피스 설계 개념

not4k의 다중키 바인딩이 만들어내는 고유한 차트 어휘. `chart-design.md`에서 각 개념의 기원, 구현 방식, 유저 경험을 정의한다.

| 이름                  | 핵심 개념                                                    |
| --------------------- | ------------------------------------------------------------ |
| 트릴 앵커             | 한 레인에 높은 밀도의 트릴로 한 손을 전담시킴                |
| 가변 분할             | 2/2 이외의 레인 분할 사용 (1/3, 3/1 등)                      |
| 레인 내 분리          | 한 레인에서 홀드+탭 동시 진행                                |
| 비트 복합 트릴        | 하나의 트릴에 8/16/24비트 등 서로 다른 비트 분할이 복합 등장 |
| 가변 분할 전환        | 곡 진행 중 레인 분할 인식이 실시간 변화                      |
| 건너가기 (크로스오버) | 내측 레인(2, 3번)에서 손의 물리적 교차 처리                  |

## 난이도 체계

### 차트 레벨 (Lv.)

채보 하나에 부여되는 난이도 수치. 밀도와 복합도의 연속적 경사로 설계된다. 노트 타입의 잠금/해금과 무관하며, 모든 노트 타입은 Lv.1부터 등장할 수 있다.

### 피스 레벨 (PLv.)

개별 피스에 부여되는 최소 활용 레벨. 해당 피스가 의미 있는 난이도 요소로 사용되기 시작하는 차트 레벨을 나타낸다. 노트 타입 자체의 등장 가능 여부(항상 Lv.1)와는 별개의 개념이다.

예: 릴리즈 교차의 PLv.가 4라는 것은 "롱노트가 Lv.4부터 등장한다"가 아니라, "롱노트가 끝날 때 키 교대를 요구하는 피스가 본격적으로 활용되는 최소 레벨이 4"라는 의미이다.

### 난이도명 (Difficulty Name)

한 곡에 배정되는 채보의 이름. 리듬게임에서는 일반적으로 한 곡에 여러 난이도의 채보가 존재하며, 각 채보에 난이도명이 붙는다 (예: IIDX의 NORMAL/HYPER/ANOTHER, DJMAX의 NM/HD/MX).

not4k의 난이도명은 미확정이다. 현재 가장 유력한 후보는 **EASY / NORMAL / HARD**의 3단계 구성.

난이도명은 차트 레벨(Lv.)과는 별개의 개념이다. 난이도명은 같은 곡 내에서 채보를 구분하는 라벨이고, 차트 레벨은 채보의 절대적 난이도 수치이다.

### 난이도 등급 (Difficulty Class)

레벨 구간을 묶어 분류한 것. 피스에도 적용된다.

| 등급   | 레벨 구간 | 키 바인딩 전제             |
| ------ | --------- | -------------------------- |
| 입문   | Lv.1~3    | 4키 주로 활용, 드물게 엄지 |
| 초급   | Lv.4~6    | 엄지 비중 상승 (6키)       |
| 중급   | Lv.7~10   | 8키 입문 (약지 추가)       |
| 상급   | Lv.11~13  | 8키 활성화                 |
| 최상급 | Lv.14~    | 8키 완전 활용, 10키 가능   |

> **용어 규칙**: 채보 단위의 난이도는 **Lv.**, 피스 단위의 난이도는 **PLv.**, 구간 분류는 **난이도 등급**으로 통일한다.

### 난이도 축 (Difficulty Axis)

레벨을 구성하는 4가지 독립적인 차원. `difficulty-design.md`에서 정의한다.

1. **노트 밀도** — 초당 입력 수 (BPM × 비트 분할)
2. **트릴 복합도** — 트릴의 길이, 속도, 다른 레인과의 동시 진행 여부
3. **더블·동시치기 복합도** — 더블 노트의 등장 위치와 빈도
4. **레인 내 복합도** — 홀드 중 탭, 릴리즈 교차 등

### 인지 부하 (Cognitive Load)

유저가 차트를 "읽는" 데 필요한 정신적 부담. 레인 분할 인식 변화, 노트 타입 복합, 레인 내 복합 등에 의해 증가한다.

### 물리 부하 (Physical Load)

유저의 손가락과 손에 가해지는 신체적 피로. 노트 밀도, 트릴 속도, 연속 더블 노트 등에 의해 증가한다.

## 판정 및 스코어링

### 판정 윈도우 (Judgment Window)

노트의 기준 시점으로부터 입력이 허용되는 시간 범위.

| 등급        | 범위  | 비고                       |
| ----------- | ----- | -------------------------- |
| **Perfect** | ±32ms | 확정                       |
| **Great**   | TBD   | 레퍼런스 수집 후 확정 예정 |
| **Good**    | TBD   | 레퍼런스 수집 후 확정 예정 |
| **Bad**     | TBD   | 레퍼런스 수집 후 확정 예정 |

### 판정 등급

| 등급    | 점수 | 설명                              |
| ------- | ---- | --------------------------------- |
| Perfect | 3    | 최고 판정                         |
| Great   | 2    |                                   |
| Good    | 1    |                                   |
| Bad     | 0    | 판정선 근처에서 입력했으나 부정확 |
| Miss    | 0    | 미입력 또는 판정 범위 초과        |

### 롱노트 판정 모델

롱노트는 헤드와 바디로 분리되어 최대 2회 판정된다.

- **헤드 판정**: 싱글 헤드는 싱글 판정 × 1, 더블 헤드는 독립된 싱글 판정 × 2.
- **바디 판정**: 두 종류가 있다. (1) **릴리즈 판정** — 뒤에 빈 공간이 있을 때, 키를 뗀 타이밍으로 판정. (2) **유지 판정** — 뒤에 다른 노트가 바로 이어질 때, 다음 노트의 틱 시점에서 홀드 유지 여부를 확인 (유지 시 Perfect, 미유지 시 Miss).
- **더블 롱노트 바디**: 독립된 싱글 바디 × 2로 병렬 판정한다. 한 키의 유지 실패가 다른 키의 판정에 영향을 주지 않는다.

### 전환 릴리즈 (Transition Release)

롱노트 연결 시 키 수가 감소하는 전환에서 발생하는 릴리즈. 키 수 감소분에 의해 릴리즈 판정 여부가 결정된다.

| 전환                  | 키 변화    | 릴리즈          |
| --------------------- | ---------- | --------------- |
| 싱글 → 더블           | 1→2 (증가) | 없음            |
| 더블 → 싱글           | 2→1 (감소) | 싱글 릴리즈 1회 |
| 더블 → 더블 (키 교체) | 2→2        | 없음 (동시)     |

### 분할 릴리즈 (Split Release)

더블 롱노트의 릴리즈를 시간차를 두고 나누어 수행하는 것. 두 손가락을 동시에 떼는 대신 하나를 먼저, 나머지를 나중에 뗀다. 표기: `D=-`

- 헤드: 더블 노트 판정 (독립된 싱글 판정 × 2)
- 첫 번째 릴리즈: 싱글 릴리즈 판정 (1회)
- 두 번째 릴리즈: 싱글 릴리즈 판정 (1회)
- 총 4회 판정.

### 릴리즈 판정

롱노트 뒤에 빈 공간이 있을 때 발생하는 바디 판정. 키를 뗀 타이밍을 기준으로 판정한다. 릴리즈 판정이 없다면 바인딩된 키를 계속 누르고 있는 것만으로 모든 롱노트를 통과할 수 있으므로, 키를 떼는 행위에 판정을 부여하여 이를 방지한다.

### 유지 판정

롱노트 뒤에 다른 노트가 바로 이어질 때 발생하는 바디 판정. 다음 노트의 틱 시점에서 홀드가 유지되고 있었는지를 확인한다. 유지 시 Perfect, 미유지 시 Miss. 홀드탭과 릴리즈 교차 모두 인정하되, 홀드를 완전히 생략하는 입력(`o-o-`를 `o.o-`로 처리)에 점수 차이를 두기 위한 설계.

### 릴리즈 판정 상향

롱노트 릴리즈 시 Good 이상이면 Perfect로 상향 보정되는 규칙. 유저의 인지 자원을 릴리즈 정밀도가 아닌 다음 노트 처리에 집중시키기 위한 설계. Bad와 Miss는 상향되지 않는다.

### TGood (트릴 오입력 Good)

트릴 노트에서 키 교대에 실패했을 때(같은 키를 연속 입력) 발생하는 특수 Good 판정. 내부 명칭 **TGood**. 일반 Good(타이밍 부정확)과 별도로 집계되어, 유저에게 "타이밍 문제"와 "손배치 문제"를 구분한 피드백을 제공한다. 결과 화면에서 별도 행으로 표시한다.

### 배드말림 (Bad Train)

더블 노트를 싱글 입력으로 처리했을 때, 미처리된 입력이 이후 노트의 판정을 연쇄적으로 밀어내는 현상. IIDX에서 유래한 용어. not4k에서는 **의도적으로 해제**한다. 더블 노트의 미처리 2번째 입력이 후속 노트를 흡수하지 않도록 하여, 더블 노트를 놓쳐도 연쇄 실패가 발생하지 않는다. 유저 경험 개선이 목적.

## 외부 참조 용어

다른 리듬게임에서 유래하여 not4k 문서에서 비교·참조 용도로 등장하는 용어.

| 용어            | 원작          | 의미                           | not4k에서의 대응                |
| --------------- | ------------- | ------------------------------ | ------------------------------- |
| 스크래치 복합   | IIDX          | 턴테이블 + 건반 동시 처리      | 트릴 앵커 피스                  |
| 노브 복합       | 사운드 볼텍스 | 아날로그 노브 + 건반 동시 처리 | 트릴 앵커 피스                  |
| Charge Note     | IIDX          | 누르고 있어야 하는 롱노트      | 롱 노트 + 키 릴레이             |
| 홀드+탭         | 츄니즘        | 슬라이더에서 홀드+탭 공존      | 레인 내 분리 피스               |
| Break 노트      | maimai        | 가중치가 높은 강조 노트        | 더블 노트의 임팩트 역할         |
| 소플란 (Soflan) | IIDX          | BPM 변화에 의한 속도 변화      | 비트 복합 트릴 피스             |
| 지력표          | IIDX 커뮤니티 | 비공식 세부 난이도표           | not4k의 공식 레벨 세분화로 대응 |
| 파지법          | IIDX          | 컨트롤러를 잡는 손 형태        | 약중검엄 배치                   |

## 표기법 심볼 요약

`piece-notation.md`에서 정의하는 수직 표기법의 심볼.

| 심볼      | 의미                                      |
| --------- | ----------------------------------------- |
| `o`       | 싱글 노트                                 |
| `t`       | 트릴 노트                                 |
| `D`       | 더블 노트                                 |
| `.`       | 빈 틱                                     |
| `-`       | 싱글 롱노트 유지 (바디)                   |
| `=`       | 더블 롱노트 유지 (바디)                   |
| `-o`      | 홀드 중 탭 (피스)                       |
| `D=-`     | 분할 릴리즈 (더블 롱노트의 시간차 릴리즈) |
| `{` / `}` | 트릴 구간 시작/종료                       |
| `~`       | 트릴 구간 내 빈 틱                        |
