# not4k — 게임 설계 개요

> 4레인 노트낙하형 가변 손배치 PC 리듬게임

이 게임이 왜 필요한지는 `rationale.md`를 참고한다.
이 문서는 게임이 **어떻게 작동하는지**를 설명한다.

---

## 핵심 메커니즘

### 4레인 + 레인당 다중키 바인딩

- 화면에는 항상 4개의 레인이 존재한다.
- 유저는 각 레인에 2개 이상의 키를 바인딩한다.
- 같은 레인의 노트는 바인딩된 키 중 아무 키로든 처리할 수 있다.
- 단, 노트 타입에 따라 특정 입력 방식이 요구될 수 있다(트릴 노트, 더블 노트).

### 가변 손배치

- 유저가 어떤 키를 어떤 손가락으로 누를지는 전적으로 유저의 선택이다.
- 차트의 복잡도에 따라 유저가 사용하는 키 수가 4개에서 8개 이상으로 확장될 수 있다.
- 레인 구조 자체는 변하지 않으므로, 키 수가 달라도 같은 난이도 체계를 공유한다.

---

## 피스 표기법

리듬게임 차트를 텍스트로 표현하기 위한 규칙이다. 아래는 간략한 수평 표기법이며, 확장 수직 표기법은 `piece-notation.md`를 참고한다.

- 간격은 백틱 블록 앞에 정의한다.
- 각 레인에 숫자로 번호를 붙인다 (1~4).
- 동시 입력은 공백 없이 붙여 쓴다.
- 각 공백은 처음 정의된 만큼의 간격을 의미한다.

**예시**:

16bit```
2 2 2 2

````

→ 2번 레인에 4개의 노트를 16비트 간격으로 배치.

16bit```
12 23 12 23
````

→ 1+2 동시치기, 2+3 동시치기가 16비트 간격으로 교대. 2번 레인은 매 비트마다 등장하므로, 고정배치에서는 연타가 되지만 가변 손배치에서는 트릴로 처리 가능.

## 노트 타입과 피스 요약

자세한 스펙은 `note-system.md`를 참고한다.

### 노트 타입

개별 요소. **모든 노트 타입은 Lv.1부터 등장할 수 있다.**

| 노트 타입     | 설명                                                                                                                | 요구하는 바인딩 키 수 |
| ------------- | ------------------------------------------------------------------------------------------------------------------- | --------------------- |
| **싱글 노트** | 레인에 바인딩된 키 중 아무 키 하나를 누른다                                                                         | 1개 이상              |
| **롱 노트**   | 키를 누르고 있다가 끝 지점에서 뗀다. 헤드+바디 구조로, 최대 2회 판정. 홀드 이어잡기로 다른 키에 홀드를 이어받을 수 있다 | 1개 이상              |
| **트릴 노트** | 트릴 구간 안에서, 직전 노트와 다른 키를 눌러야 한다                                                                 | 2개 이상              |
| **더블 노트** | 같은 레인의 서로 다른 키 2개를 동시에 누른다. 더블클릭 불가                                                         | 2개 이상              |

### 피스

복수의 노트가 배치를 통해 상호작용하여 난이도를 발생시키는 단위. 각 피스는 고유한 **PLv.(피스 레벨)**를 가진다. 정식 정의는 `piece-definition.md`의 PP-000 체계에서 관리한다.

| 피스                 | 설명                                                               | PLv.   |
| -------------------- | ------------------------------------------------------------------ | ------ |
| **릴리즈탭**         | 롱노트가 끝날 때 같은 레인에서 키 교대를 요구하는 배치             | PLv.4  |
| **싱글→더블 전환**   | 싱글 홀드에 키를 추가하여 더블 홀드로 전환 (o-D=)                  | PLv.4  |
| **트릴 앵커**        | 한 손을 트릴에 전담시키고 반대 손이 나머지 레인 처리               | PLv.7  |
| **홀드 중 탭**       | 한 키로 롱노트를 유지하면서 다른 키로 싱글 노트를 처리             | PLv.7  |
| **홀드 트릴**        | 짧은 롱노트의 빠른 연속(o-o-o-o-)으로 트릴 입력이 요구되는 배치   | PLv.7  |
| **비트 복합 트릴**   | 하나의 트릴 구간에 서로 다른 비트 분할이 복합 등장                 | PLv.10 |
| **건너가기**         | 내측 레인(2, 3번)에서 손의 물리적 교차를 요구                      | PLv.11 |
| **가변 분할**        | 2/2 이외의 레인 분할(1/3, 3/1). 건너가기 항상 동반                 | PLv.11 |
| **가변 분할 전환**   | 곡 진행 중 레인 분할이 실시간으로 변화                             | PLv.11 |

> **노트 타입 vs 피스**: 노트 타입은 개별 요소이며 Lv.1부터 등장한다. 피스는 노트들의 배치가 상호작용하여 난이도를 만드는 것이다. 예: 롱노트와 싱글노트는 각각 Lv.1부터 등장하지만, 롱노트가 끝날 때 키 교대를 요구하는 릴리즈탭 피스는 PLv.4부터 본격 활용된다. 정식 피스 카탈로그(PP-001~PP-010)는 `piece-definition.md`를 참고한다.

## 난이도 설계 요약

자세한 설계는 `difficulty-design.md`를 참고한다.

**모든 노트 타입은 가장 쉬운 레벨(Lv.1)부터 등장한다.** 난이도에 따라 변하는 것은 등장 여부가 아니라 빈도와 복합도이다.

### 난이도 체계

- **Lv. (차트 레벨)**: 차트 하나에 부여되는 난이도 수치.
- **PLv. (피스 레벨)**: 개별 피스가 본격 활용되는 최소 레벨.
- **난이도 등급**: 레벨 구간을 묶은 분류(입문/초급/중급/상급/최상급).

### 등급별 핵심 경험

| 등급       | 레벨 구간 | 유저 키 바인딩             | 핵심 경험                                                                   |
| ---------- | --------- | -------------------------- | --------------------------------------------------------------------------- |
| **입문**   | Lv.1~3    | 4키 주로 활용, 드물게 엄지 | 싱글 중심 + 내측 레인에 트릴·더블 극소량 등장으로 "이 게임은 다르다"를 인지 |
| **초급**   | Lv.4~6    | 엄지 비중 상승 (6키)       | 트릴 구간이 명확해지고, 더블이 흐름에 삽입                                  |
| **중급**   | Lv.7~10   | 8키 입문 (약지 추가)       | 트릴 앵커 본격화, 홀드 중 탭 등장                                              |
| **상급**   | Lv.11~13  | 8키 활성화                 | 건너가기·가변 분할·가변 분할 전환 등장. 모든 피스가 복합 결합               |
| **최상급** | Lv.14~15  | 16키 완전 활용             | 새 피스 없이 기존 피스의 극한 결합. 기존 4키 게임에 없는 차트 공간          |

> **용어 규칙**: 차트 단위는 **Lv.**, 피스 단위는 **PLv.**, 구간 분류는 **난이도 등급**으로 통일한다.
