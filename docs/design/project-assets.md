# not4k — 프로젝트 에셋 정의

이 문서는 not4k를 구현하기 위해 필요한 에셋을 카테고리별로 정의하고, 각 에셋의 수급 방안을 제시한다.

---

## 1. 비주얼 에셋 — 게임플레이

### 1.1 노트 그래픽

게임의 핵심 시각 요소. 각 노트 타입이 즉시 구분 가능해야 한다.

| 에셋 | 설명 | 비고 |
|------|------|------|
| 싱글 노트 | 기본 노트 스프라이트 | 4레인 × 색상/형태 변형 |
| 더블 노트 | 싱글보다 "무겁고 강조된" 느낌의 노트 | 크기 확대, 이중 테두리, 색상 강조 등 (`note-system.md`) |
| 트릴 노트 | 싱글 노트와 즉시 구분 가능한 형태 | 색상 변경, 형태 변경, 교대 방향 화살표 등 (`note-system.md`) |
| 롱노트 헤드 | 롱노트 시작점 (싱글/더블 변형) | 싱글 헤드, 더블 헤드 각각 필요 |
| 롱노트 바디 | 키 유지 구간의 연속 그래픽 | 레인 너비에 맞는 스트레처블(stretchable) 텍스처 |
| 롱노트 테일 | 롱노트 종료점 | 릴리즈/유지 판정 지점 표시 |
| 트릴 롱노트 | 트릴 구간 내의 롱노트 변형 | 트릴 노트 + 롱노트 바디 결합 |
| 더블 롱노트 | 더블 노트의 롱노트 변형 | 더블 헤드 + 바디 + 테일 |

### 1.2 트릴 구간

| 에셋 | 설명 | 비고 |
|------|------|------|
| 트릴 구간 배경 | 롱노트와 구분되는 영역 표시 | 레인 배경 색상 변경, 반투명 영역, 테두리 변화 등 (`note-system.md`) |
| 빈 트릴 구간 | 노트 없는 트릴 구간의 시각 표현 | 시각적 존재감은 유지하되 입력 요구 없음 |

### 1.3 레인 & 플레이필드

| 에셋 | 설명 | 비고 |
|------|------|------|
| 레인 배경 | 4개 레인의 배경 그래픽 | L1~L4 구분 가능한 디자인 |
| 판정선 | 노트 판정 기준선 | 하단 고정 위치 |
| 레인 구분선 | 레인 간 경계 | 가변 손배치 구간에서의 시각적 분리 |
| 플레이필드 프레임 | 전체 플레이 영역의 외곽 프레임 | |

### 1.4 판정 & 피드백

| 에셋 | 설명 | 비고 |
|------|------|------|
| 판정 텍스트 | Perfect / Great / Good / Bad / Miss | 각 등급별 색상·크기 차등 |
| TGood 표시 | 트릴 교대 실패 전용 Good 표시 | 일반 Good과 시각적으로 구분 (`scoring.md`) |
| 히트 이펙트 | 노트 적중 시 파티클/이펙트 | 판정 등급별 차등, 더블 노트 강화 이펙트 |
| 콤보 카운터 | 연속 적중 수 표시 | 숫자 + 애니메이션 |
| 풀콤보 이펙트 | Miss 없이 곡 완료 시 연출 | |

### 1.5 배경

## 7. 수급 방안

### 7.1 자체 제작

가장 많은 노력이 필요하지만 게임의 아이덴티티를 확보하는 데 필수적인 영역.

| 대상 | 이유 |
|------|------|
| 노트 그래픽 전체 (1.1) | not4k의 노트 타입(싱글/더블/트릴/롱)은 기존 리듬게임과 다른 고유 체계. 기존 에셋을 그대로 쓸 수 없음 |
| 트릴 구간 시각 (1.2) | 트릴 구간 개념 자체가 not4k 고유. 해당하는 기존 에셋이 없음 |
| 판정 이펙트 (1.4) | TGood 등 고유 판정 체계에 맞는 커스텀 이펙트 필요 |
| 로고 (2.1) | 게임 아이덴티티의 핵심 |
| 키보드 오버레이 (2.4) | 넘버패드/TKL 이중 레이아웃은 not4k 특유의 UI |
| 손 배치 다이어그램 (2.5) | 약중검엄 배치는 not4k 고유 개념 |
| 튜토리얼용 음원 (5.3) | BPM과 박자가 정확히 제어된 교육용 곡 필요 |

**필요 역량**: 2D 그래픽 디자이너 (게임 UI 경험), 사운드 디자이너 (튜토리얼 곡)

|------|--------|------|
| 로고 디자인 | Fiverr, 크몽, 숨고 | 게임 로고 전문 디자이너 |
| 노트 스킨 디자인 | 리듬게임 커뮤니티 (osu! skinners 등) | osu! 스킨 제작 경험자가 노트 그래픽 제작에 적합 |
| 히트 사운드 디자인 | Fiverr, Freesound 기반 가공 | 리듬게임 효과음 전문가 |
| 플레이용 음원 | Bandcamp, SoundCloud 아티스트 직접 교섭 | BMS 커뮤니티 작곡가 접촉 권장 |
| 자켓 이미지 | Skeb, Pixiv 리퀘스트, 크몽 | 곡별 개별 의뢰 |

### 7.5 콘텐츠 음원 수급 전략

플레이용 음원은 게임의 콘텐츠 그 자체이므로 별도 전략이 필요하다.

| 방법 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **BMS 커뮤니티 교섭** | BMS (Be-Music Source) 작곡가에게 사용 허가 요청 | 리듬게임에 최적화된 곡, 커뮤니티 친화적 | 개별 교섭 필요, 독점 사용 불가할 수 있음 |
| **CC 라이선스 음원** | Creative Commons 라이선스로 공개된 곡 활용 | 별도 교섭 불필요, 즉시 사용 가능 | 리듬게임용으로 제작된 것이 아닐 수 있음 |
| **자체 작곡** | 팀 내 작곡가 또는 전담 외주 | 게임에 완벽히 맞는 곡, 독점 사용 | 비용과 시간이 가장 많이 소요 |
| **인디 아티스트 협업** | SoundCloud, Bandcamp 아티스트와 라이선스 협상 | 다양한 장르 확보 가능 | 개별 교섭, 라이선스 조건 상이 |
| **osu! Featured Artist 모델** | 아티스트에게 노출/홍보를 제공하고 곡 사용을 허가받는 모델 | 상호 이익, 지속 가능 | 게임의 인지도가 일정 수준 이상이어야 유효 |

**초기 단계 권장**: BMS 커뮤니티 교섭 + CC 라이선스 음원으로 최소 5~10곡 확보. 튜토리얼 곡은 자체 제작 또는 단순 구조의 CC0 곡 활용.

## 8. 에셋 우선순위

구현 단계별로 필요한 에셋을 분류한다.

### Phase A: 프로토타입 (최소 플레이 가능)

게임의 핵심 메커니즘을 검증하기 위한 최소한의 에셋.

| 카테고리 | 필요 에셋 | 수급 방법 |
|----------|----------|-----------|
| 노트 | 싱글/롱/트릴/더블 노트의 플레이스홀더 그래픽 | 단순 도형(사각형, 원)으로 자체 제작 |
| 레인 | 4레인 배경, 판정선 | 단색/그라디언트로 자체 제작 |
| 판정 | 판정 텍스트 (텍스트 렌더링) | 시스템 폰트로 대체 |
| 효과음 | 히트 사운드 1종 | Freesound CC0 |
| 폰트 | 기본 UI 폰트 | Noto Sans CJK |
| 음원 | 테스트용 곡 1~2곡 | CC0 음원 또는 자체 제작 비프음 |

### Phase B: 알파 (전체 기능 동작)

모든 게임 시스템이 동작하되, 에셋 품질은 최종이 아닌 단계.

| 카테고리 | 추가 필요 에셋 | 수급 방법 |
|----------|---------------|-----------|
| 노트 | 노트 타입별 구분 가능한 디자인 | 자체 제작 또는 커미션 |
| 이펙트 | 히트 이펙트, 판정 이펙트 | 자체 제작 |
| UI | 곡 선택, 결과 화면, 키 바인딩 설정 | 자체 제작 |
| 에디터 | 타임라인 그리드, 마커, 모드 아이콘 | 자체 제작 + 오픈소스 아이콘 |
| 효과음 | 판정별 히트 사운드, UI 사운드 | Freesound + 가공 |
| 음원 | 튜토리얼 곡 + 플레이 곡 5곡 이상 | BMS 커뮤니티 + CC 음원 |

### Phase C: 베타 (품질 확정)

최종 품질의 에셋으로 교체하고 폴리싱하는 단계.

| 카테고리 | 추가 필요 에셋 | 수급 방법 |
|----------|---------------|-----------|
| 로고 | 최종 로고 | 자체 제작 또는 커미션 |
| 노트 스킨 | 최종 디자인 노트 그래픽 | 커미션 (리듬게임 스킨 디자이너) |
| 이펙트 | 풀콤보 연출, 결과 화면 애니메이션 | 자체 제작 |
| 튜토리얼 | 손 배치 다이어그램, 안내 UI | 자체 제작 |
| 음원 | 10곡 이상 라인업 확정 | 아티스트 교섭 |
| 자켓 | 곡별 자켓 이미지 | 일러스트레이터 커미션 |

## 9. 라이선스 관리 원칙

- 모든 외부 에셋은 **라이선스 출처와 조건을 기록**한다.
- CC BY 에셋은 크레딧 화면에 저작자를 표기한다.
- CC0 에셋도 출처를 기록하되, 표기 의무는 없음을 확인한다.
- 상업적 사용 불가(CC NC) 에셋은 프로토타입에서만 사용하고, 공개 빌드 이전에 교체한다.
- 폰트 라이선스는 배포 형태(웹폰트 / 임베디드)에 따라 조건이 달라질 수 있으므로 개별 확인한다.
