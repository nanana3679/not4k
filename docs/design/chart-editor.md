# not4k — 차트 편집기

이 문서는 차트 편집기의 요구사항과 차트의 데이터 모델을 정의한다.

---

## 편집기 요구사항

### 타임라인

에디터의 그리드는 다음 레인으로 구성된다.

| 레인 종류   | 개수 | 용도                          |
| ----------- | ---- | ----------------------------- |
| 노트 레인   | 4    | L1~L4 노트를 배치하는 레인   |
| BPM 레인    | 1    | BPM 마커를 배치하는 보조 레인 |
| 박자 레인   | 1    | 박자 마커를 배치하는 보조 레인 |

- 그리드 배경에 음원 파일의 **파형(waveform)** 을 표시한다.
- 파형 위에 BPM·박자 마커로부터 계산된 **박자 그리드**를 겹쳐 표시한다.
  - 그리드 라인은 마디선과 박자선을 구분할 수 있어야 한다.
  - 오프셋(ms)을 반영하여 그리드와 파형의 정렬을 맞춘다.
- 사용자는 파형과 그리드의 정렬을 시각적으로 확인하면서 **오프셋을 조절**할 수 있다.

### 줌

- 타임라인의 픽셀 밀도는 `pixelPerSecond = zoom` 으로 결정된다. zoom 값은 BPM과 무관하다.
- BPM이 변하면 시간축(픽셀)은 그대로이고 **그리드 간격이 달라진다**.
- 사용자는 **zoom** 값을 조절하여 표시되는 노트 밀도를 변경할 수 있다.

### 재생

- 음원의 **재생/정지** 기능을 제공한다.

### 스냅

- **스냅(snap)** 기능으로 그리드 눈금 간격을 조절할 수 있다. 기준 단위는 **박(beat)** 이다 (예: 1/4박, 1/8박, 1/16박, …).
- 노트 배치·이동 시 스냅 설정에 따라 가장 가까운 그리드 눈금에 흡착된다.

### 오토스크롤

- **오토스크롤** 토글을 제공한다.
- 활성화 시 재생 중 현재 재생 위치를 따라 타임라인이 자동으로 스크롤된다.

### 편집 모드

다음 세 가지 모드 중 하나를 선택하여 작업한다.

| 모드   | 단축키 | 설명 |
| ------ | ------ | ---- |
| Create | `C`    | 배치할 엔티티 종류(노트 종류·마커 종류)를 선택한 뒤 타임라인을 클릭하여 배치한다. 마커를 생성하면 직전 마커의 값을 복사한 새 마커가 생성되며, 더블클릭으로 값을 수정하는 모달이 열린다 |
| Select | `S`    | 엔티티를 선택하여 이동·삭제한다 |
| Delete | `D`    | 엔티티를 클릭하면 삭제한다. PC에서는 우클릭 삭제로 대체 가능하지만, 모바일 환경을 고려하여 별도 모드로 제공한다 |

> **이동(Move)은 별도 모드가 아닌 Select 모드의 기능으로 통합한다.** 선택된 노트를 드래그 앤 드롭하거나 방향키로 이동한다.

### 단축키

#### 모드 전환

| 키 | 동작 |
|----|------|
| `C` | Create 모드 진입 |
| `S` | Select 모드 진입 |
| `D` | Delete 모드 진입 |

#### Create 모드

| 입력 | 동작 |
|------|------|
| 타임라인 클릭 | 선택된 종류의 엔티티를 해당 위치에 배치 |
| `C` + 마우스 휠 | `C`를 누르고 있는 상태에서 휠을 굴리면 생성할 노트 타입을 변경한다. 노트 타입 드롭다운이 열린다 |

#### Select 모드

| 입력 | 동작 |
|------|------|
| 엔티티 클릭 | 해당 엔티티를 선택 (기존 선택은 해제) |
| `Shift` + 클릭 | 기존 선택을 유지한 채, 해당 엔티티를 선택에 추가 |
| `Alt` + 클릭 | 해당 엔티티를 선택에서 제거 |
| 빈 영역에서 드래그 | 드래그 영역(사각형)이 생성되며, 영역과 겹치는 노트들이 선택된다. 기존 선택은 해제된다 |
| `Shift` + 드래그 | 기존 선택을 유지한 채, 드래그 영역과 겹치는 노트들을 선택에 추가한다 |
| `Alt` + 드래그 | 드래그 영역과 겹치는 노트들을 기존 선택에서 제거한다 |
| 노트 외 영역 클릭 | 선택 해제 및 이동 확정 |
| `Enter` | 선택 해제 및 이동 확정. 배치 제약 조건을 만족하지 않으면 이동하지 않고 원래 위치로 복귀 |
| `↑` / `↓` | 선택된 노트의 이동 예정 위치를 한 틱씩 상/하로 이동 |
| 선택된 노트 드래그 앤 드롭 | 드래그 중 이동될 위치에 프리뷰를 표시하며, 배치 제약 조건 충족 여부를 시각적으로 표시한다. 드롭 시 제약 조건을 만족하면 이동 확정, 불만족 시 원래 위치로 복귀 |
| `Delete` | 선택된 노트를 삭제 |

#### 모드 무관 (공통)

| 입력 | 동작 |
|------|------|
| 노트 우클릭 | 해당 노트를 즉시 삭제 |
| 우클릭 드래그 → 노트 진입 | 우클릭을 누르고 있는 상태에서 노트 위로 드래그하면 해당 노트를 삭제. 연속 삭제에 유용 |
| `Ctrl` + 마우스 휠 | 타임라인의 zoom 값을 조절한다. 휠 업으로 확대, 휠 다운으로 축소 |

### 모바일 터치 조작

모바일 환경에서는 키보드와 마우스가 없으므로 터치 기반 대체 조작을 제공한다. 모드 전환은 화면 상단 툴바의 버튼으로 수행한다.

#### 공통

| 입력 | 동작 |
|------|------|
| 핀치 인/아웃 | 타임라인 줌 조절 |

#### Create 모드

| 입력 | 동작 |
|------|------|
| 타임라인 탭 | 선택된 종류의 엔티티를 해당 위치에 배치 |
| 노트 타입 변경 | 툴바의 노트 타입 드롭다운으로 선택 |

#### Select 모드

| 입력 | 동작 |
|------|------|
| 노트 롱프레스 | 선택 모드 진입 + 해당 노트 선택. 이후 다른 노트를 탭하면 선택에 추가/제거(토글) |
| 빈 영역에서 드래그 | 드래그 영역(사각형)이 생성되며, 영역과 겹치는 노트들이 선택된다 |
| 빈 영역 탭 | 선택 해제 및 이동 확정 |
| 선택된 노트 드래그 | 드래그 앤 드롭으로 이동. 제약 조건 불만족 시 원래 위치로 복귀 |

#### Delete 모드

| 입력 | 동작 |
|------|------|
| 노트 탭 | 해당 노트를 삭제 |

### 배치 제약 조건

편집기는 다음 제약 조건을 만족하지 않는 배치를 허용하지 않는다.

1. **동일 위치 중복 금지** — 같은 레인·같은 박자수에 두 개 이상의 노트가 존재할 수 없다.
2. **롱노트 구간 내 겹침 금지** — 롱노트 바디의 시작점과 끝점을 제외한 열린 구간 안에 다른 노트나 롱노트 시작점·끝점이 존재할 수 없다.
3. **트릴 구간 전용** — 트릴 롱노트 바디 구간 안에는 트릴 노트만 존재할 수 있고, 트릴 노트는 트릴 롱노트 바디 구간 안에만 존재할 수 있다.
4. **롱노트 시작점에 노트 필수** — 롱노트 바디 시작점에는 반드시 노트가 존재해야 한다.

### 저장 및 불러오기

- 편집 중인 차트를 서버에 **업로드**(저장)할 수 있다.
- 서버에 저장된 차트를 **로드**(불러오기)하여 편집을 이어갈 수 있다.

## 엔티티

차트를 구성하는 배치 가능한 요소. 모든 엔티티는 **위치**를 가진다.

### 위치

| 속성   | 설명                                                                 |
| ------ | -------------------------------------------------------------------- |
| 레인   | L1~L4. 노트에는 필수, 박자 마커와 BPM 마커에는 없음 (레인에 속하지 않는 전역 엔티티) |
| 박자수 | 곡 시작(0박)으로부터의 경과 박자. **분수**로 표기한다                 |

**박자수의 분수 표기**: 정수 박자는 `0`, `1`, `4`처럼 표기하고, 박자 내 세분 위치는 분수로 표기한다. 예를 들어 8비트의 두 번째 틱은 `1/2`, 16비트의 세 번째 틱은 `3/4`. 이를 통해 비트 분할에 독립적인 위치 표현이 가능하다.

---

### 노트

레인 위에 배치되는 입력 대상. 위치의 레인이 필수이다.

| 노트 종류              | 설명                                                                |
| ---------------------- | ------------------------------------------------------------------- |
| 싱글 노트              | 레인에 바인딩된 키 중 하나를 누르는 노트                            |
| 더블 노트              | 레인에 바인딩된 서로 다른 키 2개를 동시에 누르는 노트               |
| 트릴 노트              | 직전 입력과 다른 키를 눌러야 하는 노트                              |
| 싱글 롱노트 바디 시작  | 싱글 롱노트 바디 구간의 시작점. 대응하는 바디 끝을 참조한다         |
| 싱글 롱노트 바디 끝    | 싱글 롱노트 바디 구간의 끝점. 대응하는 바디 시작을 참조한다         |
| 더블 롱노트 바디 시작  | 더블 롱노트 바디 구간의 시작점. 대응하는 바디 끝을 참조한다         |
| 더블 롱노트 바디 끝    | 더블 롱노트 바디 구간의 끝점. 대응하는 바디 시작을 참조한다         |
| 트릴 롱노트 바디 시작  | 트릴 롱노트 바디 구간의 시작점. 대응하는 바디 끝을 참조한다         |
| 트릴 롱노트 바디 끝    | 트릴 롱노트 바디 구간의 끝점. 대응하는 바디 시작을 참조한다         |

**바디 시작과 끝의 상호 참조**: 롱노트 바디 시작과 바디 끝은 반드시 쌍으로 존재하며, 서로를 참조한다. 바디 시작 없는 바디 끝, 또는 바디 끝 없는 바디 시작은 유효하지 않다.

---

### 박자 마커

한 마디에 몇 박자가 들어가는지(**beatPerMeasure**)를 정의하는 전역 엔티티. 레인에 속하지 않는다. BPM이 한 박자의 시간 간격을 정의하므로, 박자 마커는 한 마디에 필요한 **박자 수만** 정의한다. beatPerMeasure는 정수 또는 분수로 표기하며, 분수는 `7/2`(=3.5박) 등 특수한 박자를 표현하기 위한 장치이다. 이 값을 통해 타임라인의 **마디선 위치가 결정**된다.

- **0박에 반드시 존재**한다. 곡의 시작 시점에서 박자가 정의되어야 한다.
- **마디가 시작하는 박자에만 배치**할 수 있다.
- 다시 등장하면 **해당 시점 이후로 beatPerMeasure가 변경**된다. 다음 박자 마커가 나올 때까지 유효하다.

예: 0박에 beatPerMeasure=`4`, 32박에 beatPerMeasure=`3` → 0~31박은 4박자 마디, 32박 이후는 3박자 마디.

---

### BPM 마커

템포를 정의하는 전역 엔티티. 레인에 속하지 않는다.

- **0박에 반드시 존재**한다. 곡의 시작 시점에서 BPM이 정의되어야 한다.
- 다시 등장하면 **해당 시점 이후로 BPM이 변경**된다. 다음 BPM 마커가 나올 때까지 유효하다.

예: 0박에 `BPM 150`, 64박에 `BPM 180` → 0~63박은 150 BPM, 64박 이후는 180 BPM.
