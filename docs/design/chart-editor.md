# not4k — 차트 편집기

이 문서는 차트 편집기의 요구사항과 차트의 데이터 모델을 정의한다.

---

## 편집기 요구사항

### 타임라인

에디터의 그리드는 다음 레인으로 구성된다.

| 레인 종류   | 개수 | 용도                          |
| ----------- | ---- | ----------------------------- |
| 노트 레인   | 4    | L1~L4 노트를 배치하는 레인   |
| BPM 레인    | 1    | BPM 마커를 배치하는 보조 레인 |
| 박자 레인   | 1    | 박자 마커를 배치하는 보조 레인 |

- 그리드 배경에 음원 파일의 **파형(waveform)** 을 표시한다.
- 파형 위에 BPM·박자 마커로부터 계산된 **박자 그리드**를 겹쳐 표시한다.
  - 그리드 라인은 마디선과 박자선을 구분할 수 있어야 한다.
  - 오프셋(ms)을 반영하여 그리드와 파형의 정렬을 맞춘다.
- 사용자는 파형과 그리드의 정렬을 시각적으로 확인하면서 **오프셋을 조절**할 수 있다.

### 줌

- 타임라인의 픽셀 밀도는 `pixelPerSecond = zoom` 으로 결정된다. zoom 값은 BPM과 무관하다.
- BPM이 변하면 시간축(픽셀)은 그대로이고 **그리드 간격이 달라진다**.
- 사용자는 **zoom** 값을 조절하여 표시되는 노트 밀도를 변경할 수 있다.

### 재생

- 음원의 **재생/정지** 기능을 제공한다.

### 스냅

- **스냅(snap)** 기능으로 그리드 눈금 간격을 조절할 수 있다 (예: 1/4, 1/8, 1/16, …).
- 노트 배치·이동 시 스냅 설정에 따라 가장 가까운 그리드 눈금에 흡착된다.

### 오토스크롤

- **오토스크롤** 토글을 제공한다.
- 활성화 시 재생 중 현재 재생 위치를 따라 타임라인이 자동으로 스크롤된다.

### 편집 모드

다음 네 가지 모드 중 하나를 선택하여 작업한다.

| 모드     | 설명                                   |
| -------- | -------------------------------------- |
| Create   | 노트·마커를 새로 배치한다              |
| Move     | 드래그 앤 드롭으로 기존 엔티티의 위치를 이동한다. 드래그 중 이동될 위치에 프리뷰를 표시하며, 배치 제약 조건 충족 여부를 시각적으로 표시한다 |
| Select   | 엔티티를 선택하여 속성을 확인·편집한다 |
| Delete   | 엔티티를 삭제한다                      |

### 배치 제약 조건

편집기는 다음 제약 조건을 만족하지 않는 배치를 허용하지 않는다.

1. **동일 위치 중복 금지** — 같은 레인·같은 박자수에 두 개 이상의 노트가 존재할 수 없다.
2. **롱노트 구간 내 겹침 금지** — 롱노트 바디 시작~끝 구간 안에 다른 노트나 롱노트 시작점·끝점이 존재할 수 없다.
3. **트릴 구간 전용** — 트릴 롱노트 바디 구간 안에는 트릴 노트만 존재할 수 있고, 트릴 노트는 트릴 롱노트 바디 구간 안에만 존재할 수 있다.
4. **롱노트 시작점에 노트 필수** — 롱노트 바디 시작점에는 반드시 노트가 존재해야 한다.

## 차트 메타데이터

하나의 차트 파일에 포함되는 곡 및 채보 정보.

| 필드       | 설명                                |
| ---------- | ----------------------------------- |
| 곡명       | 곡의 제목                                           |
| 아티스트명 | 곡의 아티스트                                       |
| 난이도명   | 채보의 난이도 라벨 (예: EASY/NORMAL/HARD)           |
| 난이도     | 차트 레벨 (Lv.) 수치                                |
| 이미지     | 곡 선택 화면 등에 표시되는 자켓 이미지              |
| 음원파일   | 곡의 오디오 파일                                    |
| 오프셋     | 음원 재생 시작으로부터 0박까지의 시간 차이 (ms 단위) |

## 엔티티

차트를 구성하는 배치 가능한 요소. 모든 엔티티는 **위치**를 가진다.

### 위치

| 속성   | 설명                                                                 |
| ------ | -------------------------------------------------------------------- |
| 레인   | L1~L4. 노트에는 필수, 박자 마커와 BPM 마커에는 없음 (레인에 속하지 않는 전역 엔티티) |
| 박자수 | 곡 시작(0박)으로부터의 경과 박자. **분수**로 표기한다                 |

**박자수의 분수 표기**: 정수 박자는 `0`, `1`, `4`처럼 표기하고, 박자 내 세분 위치는 분수로 표기한다. 예를 들어 8비트의 두 번째 틱은 `1/2`, 16비트의 세 번째 틱은 `3/4`. 이를 통해 비트 분할에 독립적인 위치 표현이 가능하다.

---

### 노트

레인 위에 배치되는 입력 대상. 위치의 레인이 필수이다.

| 노트 종류              | 설명                                                                |
| ---------------------- | ------------------------------------------------------------------- |
| 싱글 노트              | 레인에 바인딩된 키 중 하나를 누르는 노트                            |
| 더블 노트              | 레인에 바인딩된 서로 다른 키 2개를 동시에 누르는 노트               |
| 트릴 노트              | 직전 입력과 다른 키를 눌러야 하는 노트                              |
| 싱글 롱노트 바디 시작  | 싱글 롱노트 바디 구간의 시작점. 대응하는 바디 끝을 참조한다         |
| 싱글 롱노트 바디 끝    | 싱글 롱노트 바디 구간의 끝점. 대응하는 바디 시작을 참조한다         |
| 더블 롱노트 바디 시작  | 더블 롱노트 바디 구간의 시작점. 대응하는 바디 끝을 참조한다         |
| 더블 롱노트 바디 끝    | 더블 롱노트 바디 구간의 끝점. 대응하는 바디 시작을 참조한다         |
| 트릴 롱노트 바디 시작  | 트릴 롱노트 바디 구간의 시작점. 대응하는 바디 끝을 참조한다         |
| 트릴 롱노트 바디 끝    | 트릴 롱노트 바디 구간의 끝점. 대응하는 바디 시작을 참조한다         |

**바디 시작과 끝의 상호 참조**: 롱노트 바디 시작과 바디 끝은 반드시 쌍으로 존재하며, 서로를 참조한다. 바디 시작 없는 바디 끝, 또는 바디 끝 없는 바디 시작은 유효하지 않다.

---

### 박자 마커

한 마디에 몇 박자가 들어가는지(**beatPerMeasure**)를 정의하는 전역 엔티티. 레인에 속하지 않는다. 이 값을 통해 타임라인의 **마디선 위치가 결정**된다.

- **0박에 반드시 존재**한다. 곡의 시작 시점에서 박자가 정의되어야 한다.
- **마디가 시작하는 박자에만 배치**할 수 있다.
- 다시 등장하면 **해당 시점 이후로 beatPerMeasure가 변경**된다. 다음 박자 마커가 나올 때까지 유효하다.

예: 0박에 `4/4` (beatPerMeasure=4), 32박에 `3/4` (beatPerMeasure=3) → 0~31박은 4박자 마디, 32박 이후는 3박자 마디.

---

### BPM 마커

템포를 정의하는 전역 엔티티. 레인에 속하지 않는다.

- **0박에 반드시 존재**한다. 곡의 시작 시점에서 BPM이 정의되어야 한다.
- 다시 등장하면 **해당 시점 이후로 BPM이 변경**된다. 다음 BPM 마커가 나올 때까지 유효하다.

예: 0박에 `BPM 150`, 64박에 `BPM 180` → 0~63박은 150 BPM, 64박 이후는 180 BPM.
