# not4k — 문서 리뷰: 모순, 불일치, 구체적 설명 필요 사항

전체 16개 문서(설계 11개 + 리서치 5개)를 교차 검토한 결과이다.

---

## A. 문서 간 모순 / 불일치

### ~~1. 릴리즈 교차 피스의 범위 정의 불일치~~ → **해결됨**

릴리즈 교차를 "한 레인에서 롱노트가 끝날 때 키 교대를 요구하는 배치"로 통일 정의하였다. 기존의 "다른 레인 간 간섭" 개념은 제거되었으며, 릴리즈 교차는 같은 레인 내 키 교대 행위로, 릴리즈 교차 피스(PLv.4)는 이 입력을 요구하는 배치로 정의된다.

### ~~2. 분할 릴리즈(`d=-`) 판정 횟수의 모순~~ → **해결됨**

표기를 `D=-`로 변경하였다. 더블 노트 헤드(`D`)는 독립된 싱글 판정 × 2이므로, 분할 릴리즈의 판정은 헤드 2회 + 릴리즈 2회 = **총 4회**로, 더블 롱노트의 최대 판정 횟수(4회)와 일치한다.

### ~~3. Lv.1~3 등장 목록에 피스 이름 혼용~~ → **해결됨**

`note-system.md`의 주석을 수정하여, "트릴·더블"은 노트 타입의 단독 등장이고, "릴리즈 교차·홀드 중 탭"은 피스의 극소량 배치임을 명시적으로 구분하였다.

### ~~4. 더블 노트 가중치 원칙과 결론의 모순~~ → **해결됨**

`scoring.md`의 핵심 원칙을 "더블 노트는 가중치를 **가져야 한다**"에서 "더블 노트는 가중치를 **가진다**"로 수정하였다. 독립된 싱글 판정 × 2로 처리되므로 자연스럽게 2배 비중을 차지하며, 추가 보너스가 불필요하다는 설명을 추가하였다.

### ~~5. `rationale.md`의 리서치 파일 참조 경로 불일치~~ → **해결됨**

`rationale.md`의 참조 경로를 실제 파일명에 맞게 수정하였다: `iidx.md` → `beatmania.md`, `djmax-5b.md` → `djmax.md`, `sdvx.md` → `soundvoltex.md`.

### ~~6. 트릴 구간 빈 틱 입력의 모순적 서술~~ → **해결됨**

"무기능"과 "영향을 줄 수 있다"의 모순을 해소하였다. "구간 내 빈 틱에서의 입력은 구간 밖의 빈 틱과 동일하게 동작한다 — 트릴 구간이 있든 없든 결과가 같다"로 서술을 단순화하였다.

## B. 용어 불일치 / 미정의

### ~~7. "넘어가기"라는 미정의 용어 사용~~ → **해결됨**

"키 릴레이 (Hold Transfer)"를 **"홀드 이어잡기 (Hold Succession)"**로 용어를 변경하였다. `tutorial.md` Phase 3에서 잘못 병기되던 "넘어가기"는 실제로는 건너가기(크로스오버, PLv.11)의 비공식 표현이었으므로 제거하고, Phase 3의 제목을 "홀드 이어잡기"로 수정하였다. 전체 문서에서 "키 릴레이"를 "홀드 이어잡기"로 일괄 치환 완료.

### ~~8. PP-001, PP-002의 정체성 모순~~ → **해결됨**

PP-001을 노트 형태와 요구 움직임만 서술하도록 수정하고, "피스가 아닌" 주석을 제거하였다. PP-002는 문맥 설명을 제거하면 단순 더블 노트에 불과하므로 피스 카탈로그에서 삭제하였다. 이후 PP-002 결번을 제거하고 전체 번호를 재배정하였다 (PP-003→PP-002, PP-004→PP-003 등). 현재 피스 카탈로그는 PP-001~PP-010의 연속 번호 체계이다.

### ~~9. 릴리즈 교차의 이중 사용~~ → **해결됨**

릴리즈 교차를 "한 레인에서 롱노트가 끝날 때 키 교대하는 행위"로 통일 정의하였다. 피스 이름과 입력 방식 이름이 동일한 개념을 가리키게 되었으며, 피스는 "릴리즈 교차 피스"로 구분한다. 기존의 "동명 피스와의 구분" 주석은 제거되었다.

## C. 구체적 설명이 필요한 부분

### ~~10. 유예 시간 (Grace Period) 미정~~ → **해결됨**

유예 시간을 **12ms**로 확정하였다. 키 전환의 물리적 공백(5~15ms)의 상단에 해당하는 값으로, 의도적 릴리즈와 키 전환 과정의 공백을 구분한다. `note-system.md`에 수치와 근거를 명시하였다.

### ~~11. 유지 판정과 유예 시간의 상호작용 미정의~~ → **해결됨**

유지 판정 체크 시점에서 해당 레인의 모든 키가 떼어져 있더라도, 떼어진 시간이 유예 시간(12ms) 이내이면 홀드가 유지된 것으로 판정한다(Perfect). `note-system.md`에 규칙을 명시하였다.

### ~~12. 트릴 구간 내 롱 노트 배치 가능 여부 미정의~~ → **해결됨**

트릴 구간 내에 롱노트는 **트릴 롱노트**로만 배치 가능하다. `note-system.md`의 "트릴 롱노트" 섹션에서 정의: "트릴 노트는 롱노트일 수 있다. 트릴 구간 안에서 트릴 노트가 헤드가 되어 바디를 가지는 형태이다." 싱글 롱노트나 더블 롱노트는 트릴 구간 안에 배치할 수 없으며, 트릴 노트(`t`)가 헤드인 롱노트(`t-`)만 가능하다.

### ~~13. 건너가기 (크로스오버) PLv. 미정~~ → **해결됨**

PLv.11(상급)로 변경하였다. 내측 레인(2, 3번)에서만 발생하며, 1, 4번 레인은 물리적 거리로 인해 건너가기 대상에서 제외하기로 결정하였다.

### ~~14. 판정 윈도우 대부분 미정~~ → **해결됨**

Perfect ±41ms, Great ±82ms(×2), Good ±120ms(×~3), Bad ±160ms(×~4)로 전체 확정하였다. Perfect를 기준으로 대략 정수배 확장하는 배수 구조를 채택. `scoring.md`에 수치와 근거를 명시하였다. 레퍼런스는 `../research/judgment-windows.md`에 수집.

### ~~15. 난이도명 (Difficulty Name) 미확정~~ → **잠정 해결**

EASY / NORMAL / HARD의 3단계로 잠정 결정. 게임 컨셉이 구체화되면 변경 가능. 판정명(Perfect/Great/Good/Bad/Miss)도 동일하게 잠정적이다. `glossary.md`에 반영.

### 19. 용어 구조 변경 — 릴리즈 교차 삭제, 릴리즈탭/홀드 트릴 신설

- **홀드탭 → 릴리즈탭** 개명: PLv.4 피스(PP-004). 기존 릴리즈 교차의 피스 역할을 승계.

- **입력방식 섹션** 신설: `glossary.md`에 홀드 중 탭, 릴리즈탭, 홀드 이어잡기, 홀드 트릴의 4가지 입력방식을 체계적으로 정리.
- 전체 10개 문서에서 "릴리즈 교차" → "릴리즈탭", "홀드탭" → "홀드 중 탭"으로 일괄 치환.
