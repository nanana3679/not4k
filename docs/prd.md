# not4k — Product Requirements Document (PRD)

> 4레인 노트낙하형 가변 손배치 PC 리듬게임

---

## 1. 문제 정의

PC 노트낙하형 고정키 리듬게임은 세 가지 구조적 한계를 공유한다.

| # | 문제 | 근본 원인 | 영향 |
|---|------|-----------|------|
| A | **연타의 물리적 고통** | 한 키에 한 손가락이 고정되어 반복 충격이 집중 | 건초염·손목 터널 증후군, 고난이도 패턴 단순화 |
| B | **키 수 간 난이도 단절** | 4B→5B→6B→8B 전환 시 레인 구조가 변경 | 키 수마다 사실상 다른 게임, 통합 성장 경로 부재 |
| C | **고정 손배치의 해부학적으로 불리한 패턴** | 레인-키 1:1 고정으로 해부학적 한계에 충돌. 키 수가 늘수록(5B·7+1) 심화 | IIDX 4756 문제, 랜덤 의존, 실력이 아닌 운에 의한 클리어 |

아케이드 리듬게임(SDVX, 츄니즘, maimai)은 물리적 인터페이스 특성으로 이 문제를 부분적으로 해결하지만, PC 키보드 환경으로 이식할 수 없다.

**근거 문서**: `rationale.md`, `docs/research/` (6개 역기획 보고서)

## 2. 해법

### 핵심 메커니즘: 4레인 + 레인당 다중키 바인딩

- 화면에는 항상 4개의 레인이 존재한다.
- 유저는 각 레인에 **2개 이상의 키**를 바인딩한다.
- 같은 레인의 노트는 바인딩된 키 중 아무 키로든 처리할 수 있다.

| 기존 문제 | 해법 | 효과 |
|-----------|------|------|
| 연타의 고통 | 연타를 트릴(키 교대)로 처리 가능 | 반복 충격 분산, 패턴 다양성 확보 |
| 키 수 단절 | 레인 수 불변, 키 수만 유저가 확장 | 4키→16키가 하나의 연속된 성장 경로 |
| 해부학적으로 불리한 패턴 | 어떤 키 조합을 쓸지 유저가 선택 | 해부학적 한계가 시스템에 의해 강제되지 않음 |

**트레이드오프**: 입력 난이도 감소 대신 인식 난이도(어떤 키로 처리할지의 실시간 판단) 증가. 트릴 노트와 난이도 계단으로 관리.

## 3. 타겟 유저

### 1차 타깃: 기존 노트낙하형 리듬게임 유저 중 신체적 부하와 물리적 한계로 흥미를 잃은 유저

osu!mania, DJMAX 등 4키+ 리듬게임 경험자 중 고난이도에서 **신체적 부하와 물리적 한계**로 흥미를 잃은 유저.

| 기존 4키의 문제 | not4k의 해법 |
|---------------|-------------|
| 물리적 속도·지구력이 난이도 천장 | 신체 부하 대신 **인식 부하**(손배치 전환 판단)로 난이도 상승 |
| 특정 손가락에 부하 집중 (약지·소지) | 8키 분산 → 손가락 사용이 **균등하게 분산** |
| 배치가 불편해도 강제 | 원하는 손가락을 선택 가능 → **배치 스트레스 없음** |

### 2차 타깃: 리듬게임 입문자

리듬게임을 처음 접하는 유저도 not4k로 시작할 수 있도록 **진입 장벽을 낮추는 장치**를 마련한다:

- 튜토리얼 (Phase B): 연주 방법 → 복합 패턴 단계적 학습
- Lv.1~3: 전 노트 타입이 등장하지만, 패턴 밀도와 조합이 단순
- 옵저버 모드 (Phase B): 어려운 구간을 미리 연습 가능

단, **메인 타깃은 신체적 부하와 물리적 한계로 기존 리듬게임에서 흥미를 잃은 유저**이다. 입문자는 "이 게임으로도 시작할 수 있다"는 접근성을 보장하되, 게임 설계의 핵심 의사결정은 1차 타깃 기준으로 내린다.

### 3차 타깃: 상급 유저

고난이도 도전과 랭킹 경쟁을 원하는 유저. 기존 게임에 없는 차트 공간(홀드 중 탭, 건너가기 등)이 동기 부여.

### 지역 타깃

한국 → 일본 → 글로벌 순. 다국어(ko/ja/en) 확정.

## 5. 노트 시스템

### 5.1 노트 타입

모든 노트 타입은 **Lv.1부터 등장**. 난이도에 따라 변하는 것은 빈도와 복합도.

| 노트 타입 | 메커니즘 | 요구 키 수 | 판정 |
|-----------|----------|------------|------|
| **싱글** | 레인 내 아무 키 1개 | 1+ | 1회 |
| **롱** | 키 누르기 → 유지 → 떼기. 헤드+바디 구조 | 1+ | 최대 2회 (헤드+릴리즈) |
| **트릴** | 직전 키와 **다른 키**. 트릴 구간 내에서만 등장 | 2+ | 1회 (교대 실패 시 Good◇) |
| **더블** | 서로 다른 키 **2개 동시** | 2+ | 독립 싱글 판정 × 2 |

### 5.2 특수 메커니즘

| 메커니즘 | 설명 |
|----------|------|
| **홀드 이어잡기** | 롱노트 중 다른 키로 홀드 전환. 유예 시간 12ms 이내 릴리즈는 유지로 간주 |
| **릴리즈 판정 상향** | 바디 판정 시 Good 이상은 Perfect로 상향. 바디 결과는 Perfect/Bad/Miss 삼분법 |
| **더블-싱글 혼합 배드말림 방지** | 더블 노트의 미처리 2번째 슬롯이 후속 싱글 노트를 흡수하지 않음. 일반 배드말림(이른 입력에 의한 가장 이른 노트 매칭)은 정상 동작 |
| **바디 구간 독립성** | `o-o-`의 각 바디 구간은 독립 판정 |
| **트릴 구간** | 시각적 예고 영역. 입력 불요구. 긴장도 유지·시각적 다양성 역할 |

### 5.3 피스 (노트 배치의 상호작용)

| 피스 | PLv. | 설명 |
|------|------|------|
| 짧은 트릴 | 1 | 트릴 노트가 2~4개만 등장하는 최소 트릴 구간 |
| 릴리즈탭 | 4 | 롱노트 릴리즈 + 같은 레인 키 교대 |
| 싱글→더블 전환 | 4 | 싱글 노트 흐름이 더블 노트로 전환되는 배치 |
| 홀드 중 탭 | 7 | A키 홀드 유지 + B키로 싱글 노트 |
| 홀드 트릴 | 7 | 짧은 롱노트의 빠른 연속 (`o-o-o-o-`) |
| 앵커 | 7 | 한 손 전담 트릴 + 반대 손 복합 배치 |
| 비트 복합 트릴 | 10 | 하나의 트릴에 여러 비트 분할이 복합 |
| 건너가기 | 11 | 내측 레인에서 손의 물리적 교차 |
| 가변 분할 | 11 | 2/2 이외의 레인 분할. 건너가기 항상 동반 |


**정식 피스 카탈로그**: `piece-definition.md` (PP-001~PP-010)

## 6. 난이도 체계

Lv.1~15, 5단계 등급.

| 등급 | 레벨 | 키 바인딩 | 핵심 경험 |
|------|------|-----------|-----------|
| **입문** | 1~3 | 4키 주로, 드물게 엄지 | 4레인 인식. 싱글 90%↑, 트릴·더블 극소량 |
| **초급** | 4~6 | 엄지 비중 상승 (6키) | 트릴 구간 명확화, 릴리즈탭(PLv.4) 본격화 |
| **중급** | 7~10 | 8키 입문 (약지 추가) | 앵커 본격화, 홀드 중 탭(PLv.7) 등장 |
| **상급** | 11~13 | 8키 활성화 | 건너가기·가변 분할 등장. 전 피스 고밀도 복합 결합 |
| **최상급** | 14~15 | 16키 완전 활용 | 새 피스 없이 극한 결합. 기존 4키 게임에 없는 차트 공간 |

**등급 간 전이 설계**: 온보딩 차트(새 피스 저밀도 도입), 역방향 동기 부여(키 추가 후 같은 곡 재시도), 난이도 표시(Lv. + 등급 명시).

## 7. 스코어링

| 항목 | 규격 |
|------|------|
| **달성률** | (획득 점수 / 최대 가능 점수) × 100% |
| **점수** | Perfect=3, Great=2, Good=1, Good◇=1, Bad=0, Miss=0 |
| **콤보 파괴** | Bad, Miss |
| **풀콤보** | Miss + Bad = 0 (Good◇은 풀콤보에 영향 없음) |
| **랭크** | SSS(≥99.5%) ~ F(<60%) 10단계 |
| **게이지** | 없음. 무조건 완주, 게임 오버 없음 |

## 8. 키 바인딩

### 8.1 손배치 철학

**약중검엄**: 약지-중지-검지-엄지의 4손가락. 소지 배제(힘줄 간섭, 약한 근력, 자세 비틀림).

### 8.2 진입점 배치 (8키)

8키는 레인당 2키의 최소 바인딩으로, 모든 노트 타입을 처리할 수 있는 진입점이다. 기본 프리셋(16키)의 상세 배치는 `keybinding.md`를 참고한다.

| 레인 | 왼손 (손가락) | 오른손 — 넘버패드 | 오른손 — TKL |
|------|---------------|-------------------|--------------|
| 1 | Q(약지) + W(중지) | — | — |
| 2 | E(검지) + C(엄지) | — | — |
| 3 | — | 7(검지) + 1(엄지) | P(검지) + ,(엄지) |
| 4 | — | 8(중지) + 9(약지) | [(중지) + ](약지) |

기본값: **TKL 배치** (게이밍 키보드 다수가 텐키리스).

### 8.3 키 규칙

- 레인당 2키 이상 바인딩 필수 (최소 8키). 기본 프리셋은 레인당 4키 (총 16키)
- 바인딩된 키는 모두 동등 (주키/보조키 구분 없음)
- 하나의 물리 키는 하나의 레인에만 바인딩 (충돌 불가)

## 9. 튜토리얼

4단계 구조. 키 바인딩 발견(`keybinding.md`)과 노트 메커니즘 학습(`tutorial.md`)을 병행.

| Phase | 목표 | 소개하는 것 | 소개하지 않는 것 |
|-------|------|-------------|------------------|
| 1 | 모든 노트 타입을 처리할 수 있다 | 싱글, 롱, 트릴, 더블의 기본 입력 | 판정 세부 규칙 |
| 2 | 판정 결과를 이해하고 개선 방향을 안다 | 릴리즈 vs 유지 판정, Good◇, 분할 릴리즈 | — |
| 3 | 롱노트 중 키 전환이 가능함을 체득 | 홀드 이어잡기, 유예 시간 | — |
| 4 | 16키 바인딩의 이점 이해 | 3~4키 트릴, 더블홀드+트릴 처리 | — |

## 10. 기술 요구사항

### 10.1 플랫폼

| 항목 | 결정 | 근거 |
|------|------|------|
| **배포** | 웹 (브라우저) | URL 클릭 → 즉시 플레이. 설치 불필요, OS 무관 |
| **온라인** | 온라인 전용 | 음원 스트리밍(저작권 비용), 차트 서버 관리, 랭킹 |
| **오프라인** | 미지원 | 로컬 캐싱·동기화·충돌 해결 복잡도 회피 |
| **게임패드** | 미지원 | 트릴 교대에 부적합한 버튼 배치 |
| **프레임레이트** | 60/120/144fps/무제한 선택 | 유저 하드웨어 대응. 판정은 프레임 비의존 |

### 10.2 인증

| 항목 | 결정 |
|------|------|
| 소셜 로그인 | Google (단일 제공자로 시작) |
| 비로그인 | 플레이 가능. 기록·랭킹·설정 동기화 불가. 설정은 localStorage에 로컬 저장 |

### 10.3 오프셋

| 오프셋 | 용도 |
|--------|------|
| **오디오 오프셋** | 출력 장치(스피커/이어폰/블루투스) 지연 보정 |
| **입력 오프셋** | 입력 장치(키보드) 지연 / 개인 체감 보정 |

두 오프셋은 원인이 다르므로 분리. 전용 캘리브레이션 화면 제공. 오디오 동기화는 AudioContext 마스터 클럭 기반으로 최소 지연 보장, 나머지는 캘리브레이션으로 보정.

### 10.4 에러 처리

| 시나리오 | 동작 |
|----------|------|
| 플레이 중 네트워크 단절 | 음원/차트는 사전 로드 완료이므로 플레이 계속. 결과 전송은 복구 후 재시도 |
| 음원 로딩 실패 | 최대 3회 자동 재시도 후 실패 시 에러 표시 + 곡 선택 복귀 |
| 서버 점검 | 로그인 여부 무관 전체 차단. 점검 안내 화면 표시 |

## 11. 에셋 전략

### 왜 무료로 시작하는가

초기 자본이 없으므로 음원을 구매할 수 없다. 대신 동인(BMS) 작곡가들이 **비상업 목적 사용을 허가**한 곡을 활용하여 게임을 공개한다. 비상업 단계에서 재미를 검증하고 유저를 확보한 뒤, 수요가 확인되면 이미 커넥션이 형성된 작곡가들과 **상업 라이선스 계약으로 전환**한다. 비상업 단계의 협력 관계가 상업화 전환 비용을 낮추는 핵심 자산이다.

### 단계별 계획

| 단계 | 목표 곡수 | 에셋 수급 | 비용 |
|------|----------|-----------|------|
| **Phase A: 프로토타입** | 3~5곡 | 플레이스홀더(단순 도형), CC0 효과음, Noto Sans CJK | 0원 |
| **Phase B: 알파 (비영리 공개)** | 5~10곡 | 노트 타입별 디자인 자체 제작, BMS 작가 비상업 허가 교섭, Freesound 가공 | 0~30만원 |
| **Phase C: 베타** | 10곡+ | 최종 노트 스킨 커미션, 로고, 자켓 의뢰 | 10~100만원 |
| **Phase D: 상업화 (선택)** | — | 기존 BMS 작가와 상업 라이선스 계약 전환, CC-NC 곡 교체, 오리지널 곡 커미션, 법률 검토 | 별도 |

**음원 확보**: BMS 작가 비상업 허가(핵심), CC 라이선스 곡, Newgrounds/SoundCloud CC 필터, 자체 작곡, 작곡 콘테스트.

**라이선스 원칙**: 모든 외부 에셋 출처·조건 기록, CC-NC 에셋에 `non-commercial-only` 태그, BMS 작가 허가 서면 보존.

**근거 문서**: `project-assets.md`
