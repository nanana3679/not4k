# not4k — 스코어링 시스템

> 판정 윈도우, 풀콤보 조건, 랭크 시스템이 확정되었다. 실력 지표(볼포스류)는 미정.

---

## 설계 방향

### 참고하는 기존 모델

| 게임              | 스코어링 특징                                                                                          | not4k에의 시사점                                                              |
| ----------------- | ------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------- |
| **maimai**        | Break 노트가 전체 달성률 101% 중 1%를 독점. Critical Perfect 내에서도 타이밍 정밀도에 따라 보너스 차등 | 더블 노트에 가중치를 부여하여 "반드시 정확하게 쳐야 하는 순간"을 만들 수 있음 |
| **츄니즘**        | Justice Critical(101% 가중치)과 ExTAP(판정 관대)의 이원 구조                                           | 트릴 노트의 판정을 관대하게 설정하여 학습 과정의 좌절감을 줄일 수 있음        |
| **IIDX**          | PGreat/Great/Good 판정 + 게이지(Normal/Hard/Ex-Hard) 이원 구조                                         | 게이지 시스템으로 "생존"과 "정확도"를 분리하여 다양한 플레이 목표를 제공      |
| **사운드 볼텍스** | 1000만점 만점 절대 점수제 + 볼포스(상위 50곡 기반 동적 수치)                                           | 유저 실력의 정량화 수단으로 볼포스 같은 집계 시스템을 검토할 수 있음          |

### 핵심 원칙

1. **더블 노트는 가중치를 가진다**: 더블 노트는 독립된 싱글 판정 × 2로 처리되므로, 자연스럽게 싱글 노트의 2배 비중을 차지한다. 별도의 보너스 없이도 더블 노트를 정확하게 입력해야 할 동기가 발생한다.

2. **트릴 노트의 판정은 학습을 돕는 방향으로**: 트릴 노트는 가변 손배치의 학습 장치이므로, 최소한 초급 등급(Lv.4~6)에서는 판정이 관대해야 한다.

3. **입문 등급(Lv.1~3)은 기존 4K와 동일한 스코어링 감각**: 기존 4K 유저가 위화감 없이 진입할 수 있어야 한다.

## 판정 윈도우

노트의 기준 시점으로부터 입력이 허용되는 시간 범위.

| 등급        | 범위   | 배수       |
| ----------- | ------ | ---------- |
| **Perfect** | ±41ms  | 기준       |
| **Great**   | ±82ms  | Perfect×2  |
| **Good**    | ±120ms | Perfect×~3 |
| **Bad**     | ±160ms | Perfect×~4 |

### 판정 윈도우의 근거

**Perfect ±41ms를 선택한 이유**: 기존 PC 리듬게임의 최고 판정은 IIDX PGreat ±16.67ms, DJMAX 100% ±22ms(추정), osu!mania 300g ±16ms 등으로 매우 좁다. not4k는 가변 손배치와 트릴 교대라는 추가 인지 부하가 존재하므로, 순수 타이밍 정밀도에서는 기존 게임보다 여유를 두는 것이 적절하다. ±41ms는 IIDX PGreat의 약 2.5배, DJMAX 100%의 약 1.9배로, 인지 부하 증가분을 보상하면서도 타이밍 감각의 의미를 유지하는 범위이다.

**배수 구조를 채택한 이유**: Great ±82ms(×2), Good ±120ms(×~3), Bad ±160ms(×~4)로, Perfect를 기준으로 대략 정수배 확장하는 구조이다. 이 구조는 유저에게 직관적인 난이도 감각을 제공한다 — "Perfect의 두 배 범위가 Great"라는 관계가 명확하다. Good과 Bad는 정확한 정수배가 아니지만, 주요 리듬게임(IIDX Great ±33.33ms = PGreat×2, osu!mania 200 ±127ms ≈ 300g×8)의 사례를 참고하여 게임플레이에 적합한 범위로 조정하였다.

**상세 레퍼런스**: `../research/judgment-windows.md`에 IIDX, SDVX, DDR, PIU, EZ2ON, DJMAX, osu!, 프로젝트 세카이, Arcaea의 판정 윈도우를 수집·비교하였다.

## 노트 타입별 판정 규칙

### 싱글 노트

판정 윈도우 내 입력 시 해당 등급 부여. 노트가 판정 윈도우를 완전히 지나간 경우 Miss. 이른 입력(노트가 윈도우에 도달하기 전의 입력)은 Miss를 발생시키지 않으며, 가장 가까운 노트의 Bad 판정으로 처리된다.

### 더블 노트

**독립된 싱글 판정 × 2**로 처리한다. 두 입력은 개별적으로 판정되며, 각각 다른 판정 등급을 받을 수 있다 (예: 첫 번째 입력이 Perfect, 두 번째 입력이 Great). 서로 다른 키 2개를 동시에 눌러야 하며, 같은 키의 더블클릭은 인정되지 않는다.

### 롱노트

롱노트는 시작점·바디·끝점으로 구성되며, 끝점에서 1회 판정(종결 또는 연결)을 받는다. 헤드는 시작점에 일반 노트를 별도로 배치하여 구현하며, 헤드 노트의 판정은 해당 노트 타입의 규칙을 따른다.

#### 헤드 노트 판정

헤드 노트는 롱노트와 별개의 엔티티이므로, 해당 노트 타입의 판정 규칙이 독립적으로 적용된다.

- 싱글 노트 헤드: 싱글 판정 × 1.
- 더블 노트 헤드: 독립된 싱글 판정 × 2.
- 헤드 노트가 없으면(롱노트만 단독 존재) 헤드 판정은 0회.

#### 바디 판정

롱노트의 판정은 **유지 판정**과 **끝점 판정** 두 단계로 구성된다.

**유지 판정**: 바디 구간 동안 해당 레인에 held 상태가 지속되는지 감시한다. 키의 held 상태는 **키 단위**로 추적한다. 유예 시간(Grace Period, 12ms)보다 길게 해당 레인의 모든 키가 릴리즈되면 **즉시 유지 실패 상태로 전환**된다. 이 실패는 복구할 수 없다 — 유예 시간 초과 후 다시 키를 눌러도 해당 롱노트의 판정은 끝점에서 Miss로 확정된다. 유예 시간 이내의 짧은 릴리즈(홀드 이어잡기 등)는 허용된다. 단, 시작점·끝점 근처 Good 윈도우 구간에서는 유지 판정이 완화된다: 시작점 이후 +120ms 이내의 키 미입력, 끝점 이전 −120ms 이내의 릴리즈는 유지 실패로 처리하지 않는다. 유지 판정 시작 조건은 헤드 노트 유무와 무관한 단일 규칙이다.

바디 판정의 결과는 **Perfect, Bad, Miss**의 세 가지이다. Good 이상의 유지/릴리즈는 Perfect로 상향되고, Bad는 그대로 Bad, 유지 실패는 Miss이다. Great/Good은 발생하지 않는다. not4k는 다중키 바인딩, 트릴, 홀드 이어잡기 등으로 이미 충분한 인지적 부하를 제공하므로, 릴리즈 타이밍 정밀도에 대한 요구를 완화하는 구조이다.

**끝점 판정**: 싱글 롱노트는 끝점에서 **1회의 판정**만 가진다. 이 판정은 끝점에 같은 레인에 다른 **롱노트의 시작점**이 존재하는지 여부에 따라 **연결 판정** 또는 **종결 판정** 중 하나가 적용된다. 두 판정은 상호 배타적이다. 바디 유지에 실패한 경우, 종류에 관계없이 Miss이다. 끝점 판정의 트리거는 이벤트 기반이다: 끝점 Good 윈도우 내에서 릴리즈가 발생하면 릴리즈 시점에, 끝점까지 홀드를 유지하면 끝점에서 판정이 촉발된다.

**종결 판정**: 끝점에 같은 레인에 다른 롱노트의 시작점이 없을 때 적용된다(싱글/더블/트릴 노트만 있거나 아무 노트도 없는 경우 포함). 릴리즈 타이밍과 끝점 타이밍의 차이를 기준으로 판정하되, Good 이상은 Perfect로 상향된다. Good 윈도우(±120ms) 이후에 릴리즈가 발생하면 Bad, 판정 윈도우(Bad, ±160ms)를 완전히 벗어나면 Miss. 바디 유지 실패 시 Miss.

**연결 판정**: 끝점에 같은 레인에 다른 롱노트의 시작점이 존재할 때 적용된다. 판정 촉발 시점에 해당 레인에 홀드가 유지되고 있는지를 확인한다. 유지 시 Perfect, 미유지 시 Miss.

- 예: `o-o-`에서 첫 번째 롱노트의 끝점과 두 번째 롱노트의 시작점이 같은 박자수이므로, 첫 번째 롱노트는 연결 판정을 받는다. 홀드 중 탭(`o-o-`를 `a---` + `..b.`로 처리)과 홀드 이어잡기(`a-b-`로 키를 교대하여 이어받기) 모두 유효하며, 홀드를 완전히 생략하는 입력(`o.o-`)에만 점수 차이가 발생한다.
- 예: `o-o`(롱노트 끝 + 싱글 노트, 후속 롱노트 없음)에서 롱노트는 종결 판정을 받는다. 유저는 유지 키를 떼면서 다른 키로 싱글 노트를 처리한다(릴리즈탭).

**바디 구간의 독립성**: `o-o-`처럼 노트를 경유하여 바디가 이어지는 경우, 각 바디 구간은 독립적으로 판정된다. 첫 번째 바디 유지에 실패하더라도 두 번째 `o`에서 다시 홀드하면 두 번째 바디는 정상 판정된다.

#### 더블 롱노트 바디

더블 롱노트의 바디는 2키를 동시에 유지해야 하며, 각 키의 홀드 상태는 독립적으로 추적된다. 1키 릴리즈 시 해당 키의 롱노트는 즉시 유지 실패가 되며 다시 잡을 수 없다. 나머지 1키는 싱글 롱노트처럼 독립적으로 판정된다. 한 키의 유지 실패가 다른 키의 판정에 영향을 주지 않는다.

#### 롱노트 연결 시 키 수 변화와 릴리즈

롱노트 연결 시 키 수가 변하는 전환에서, 종결 판정 여부는 **키 수 감소분**에 의해 결정된다.

| 전환                  | 키 변화    | 릴리즈          |
| --------------------- | ---------- | --------------- |
| 싱글 → 더블 (`o-D=`)  | 1→2 (증가) | 없음            |
| 더블 → 싱글 (`D=o-`)  | 2→1 (감소) | 싱글 릴리즈 1회 |
| 더블 → 더블 (키 교체) | 2→2        | 없음            |

#### 분할 릴리즈 (Split Release)

더블 롱노트의 릴리즈를 시간차를 두고 나누어 수행하는 것. 표기: `D=-`

- 헤드: 더블 노트 판정 (독립된 싱글 판정 × 2)
- 첫 번째 릴리즈: 싱글 종결 판정 (1회)
- 두 번째 릴리즈: 싱글 종결 판정 (1회)
- **총 4회 판정.**

### 트릴 노트

#### 교대 성공 시

일반 판정 규칙과 동일. 판정 윈도우에 따라 Perfect/Great/Good/Bad/Miss.

#### 교대 실패 시 (Good◇)

트릴 노트에서 **직전에 입력한 키와 동일한 키**를 입력한 경우(교대 실패), **판정은 Good으로 고정된다.**

교대 실패의 정의: 직전에 사용한 키와 같은 키를 다시 누르는 것이다. 바인딩된 키가 3개 이상이더라도, 직전 키만 아니면 어떤 키를 눌러도 교대 성공이다. 예를 들어 키 A, B, C가 바인딩된 레인에서 `A → B → C`는 교대 성공이고, `A → B → B`는 두 번째 B에서 교대 실패이다.

이것은 일반적인 Good 판정과 별도로 집계한다. 결과 화면에서 Good◇의 횟수를 별도 행으로 표시하여, 유저에게 트릴 처리에 대한 구체적인 피드백을 제공한다.

설계 의도:

- **Miss가 아닌 Good인 이유**: 트릴 노트는 가변 손배치의 학습 장치이다. 교대에 실패했더라도 타이밍 자체는 맞았으므로, 완전한 실패(Miss/Bad)로 처리하면 학습 의욕이 꺾인다. 1점을 부여하되 Perfect/Great보다는 명확히 낮은 점수를 줌으로써, "맞추긴 했지만 개선의 여지가 있다"는 피드백을 전달한다.
- **별도 집계하는 이유**: 일반 Good과 Good◇은 원인이 다르다. 일반 Good은 타이밍 부정확이고, Good◇은 손배치 미숙이다. 유저가 자신의 약점이 "타이밍"인지 "손배치"인지를 구분할 수 있어야 실력 향상의 방향을 잡을 수 있다.

### 배드말림 (Bad Train)

더블 노트를 싱글 입력으로 처리했을 때, 미처리된 입력이 이후 노트의 판정을 연쇄적으로 밀어내는 현상. **의도적으로 해제한다.** 더블 노트의 미처리 2번째 입력이 후속 노트를 흡수하지 않도록 하여, 더블 노트를 놓쳐도 연쇄 실패가 발생하지 않는다. 유저 경험 개선이 목적이며, 미처리에 대한 페널티는 해당 노트의 판정(1개만 성공)으로 충분하다. 상세한 메커니즘은 `note-system.md`의 더블 노트 섹션을 참고.

### 콤보와 풀콤보

#### 콤보 파괴 규칙

| 판정 등급   | 콤보   |
| ----------- | ------ |
| **Perfect** | 유지   |
| **Great**   | 유지   |
| **Good**    | 유지   |
| **Good◇**   | 유지   |
| **Bad**     | **파괴** |
| **Miss**    | **파괴** |

Bad는 콤보를 파괴한다. 판정선 근처까지 도달했지만 부정확한 입력이므로, 콤보 유지의 가치를 갖지 못한다.

#### 풀콤보 (Full Combo)

**Miss와 Bad가 없으면 풀콤보이다.** 헤드, 바디, 트릴 등 모든 판정에서 Miss와 Bad가 한 번도 발생하지 않아야 한다.

- 바디 판정이 Perfect/Bad/Miss 삼분법이므로, 바디의 Bad 또는 Miss도 풀콤보를 깨뜨린다.
- Good◇(트릴 교대 실패)은 Bad도 Miss도 아니므로, 풀콤보에 영향을 주지 않는다. 교대에 실패했더라도 타이밍 자체는 맞았으므로, 풀콤보를 깨뜨리는 것은 과도하다.

**풀콤보 조건에 Bad를 포함한 이유**: Bad(±160ms)는 기준 시점에서 상당히 벗어난 입력이다. 풀콤보가 "정확한 연주의 증거"로 기능하려면, 콤보를 파괴하는 수준의 부정확한 입력이 없어야 한다. Good(±120ms)까지는 허용하되 Bad부터는 풀콤보를 불허함으로써, 풀콤보의 달성 가치를 유지한다.

## FAST/SLOW 표시

노트 판정 시 유저가 기준 시점보다 빨리(FAST) 또는 늦게(SLOW) 입력했는지를 표시한다.

### 정의

- `|deltaMs| > Perfect/2 (20.5ms)` 이고 `deltaMs < 0` → **FAST** (기준 시점보다 빨리 입력)
- `|deltaMs| > Perfect/2 (20.5ms)` 이고 `deltaMs > 0` → **SLOW** (기준 시점보다 늦게 입력)
- `|deltaMs| ≤ Perfect/2 (20.5ms)` → 표시 없음 (Perfect 윈도우 내측 절반)

### 대상

- **유저 입력이 있는 헤드 판정만**: 싱글, 더블, 트릴 노트의 헤드 판정에서 deltaMs가 존재할 때 표시한다.
- **비대상**: Miss(자동 — 유저 입력 없음), 바디 판정(릴리즈/홀드 기반이므로 FAST/SLOW 의미 없음).

### 표시

- 플레이 중 판정 텍스트 아래에 "FAST" 또는 "SLOW"를 색상과 함께 표시한다. FAST는 파란색 계열, SLOW는 빨간색 계열.
- 결과 화면에서 FAST/SLOW 횟수를 각각 표시한다.
- 설정 화면에서 FAST/SLOW 표시 여부를 토글할 수 있다 (기본값: ON).

## 달성률과 랭크

### 달성률 (Achievement Rate)

달성률은 유저의 연주 정확도를 백분율로 나타낸 값이다.

```
달성률 = (획득 점수 합 / 최대 가능 점수) × 100%
```

- 최대 가능 점수: 모든 판정이 Perfect일 때의 총점. 각 Perfect = 3점.
- 획득 점수: 실제 판정 결과의 점수 합. Perfect = 3, Great = 2, Good = 1, Good◇ = 1, Bad = 0, Miss = 0.

### 랭크 (Rank)

| 랭크  | 달성률 기준 |
| ----- | ----------- |
| **SSS** | ≥ 99.5%   |
| **SS**  | ≥ 99.0%   |
| **S**   | ≥ 97.0%   |
| **AAA** | ≥ 95.0%   |
| **AA**  | ≥ 90.0%   |
| **A**   | ≥ 85.0%   |
| **B**   | ≥ 80.0%   |
| **C**   | ≥ 70.0%   |
| **D**   | ≥ 60.0%   |
| **F**   | < 60.0%    |

**10단계 랭크를 채택한 이유**: SSS~F의 10단계는 유저의 성장 과정에서 충분한 중간 목표를 제공한다. SSS(99.5%)는 사실상 올퍼펙에 가까운 달성도이며, F(60% 미만)는 차트를 완주했지만 대부분의 노트를 놓친 상태를 나타낸다. 90%대에 AAA/AA/A 세 단계를 두어, 중급~상급 유저의 미세한 성장도 가시적으로 반영한다.

**랭크 산출은 달성률 단일 기준**: 게이지가 없고 무조건 완주이므로, 클리어/실패 구분이 불필요하다. 달성률 하나만으로 랭크가 결정되며, 시스템이 단순하고 직관적이다.

## 미정 사항

### 실력 지표 `[미구현]`

- 사운드 볼텍스의 볼포스처럼, 유저의 전체 실력을 하나의 수치로 집계하는 시스템 도입을 검토 중이다.
- 볼포스류 시스템은 난이도가 정확히 책정되어야 의미가 있으므로, 충분한 플레이 데이터가 축적된 이후에 난이도를 조정하면서 도입한다.
- 단위인정(段位認定)은 볼포스 시스템이 실력 검증 기능을 대체하므로 도입하지 않는다.
