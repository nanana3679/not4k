# not4k — 곡 프리뷰 구간 설정

이 문서는 곡 업로드 시 음원의 프리뷰 구간을 지정하고, 해당 구간만 잘라서 프리뷰 오디오로 저장하는 기능을 정의한다.

---

## 배경

현재 곡 추가(Add Song) 모달에서는 프리뷰 오디오를 **별도의 파일**로 업로드한다. 이 방식은 사용자가 외부 도구(Audacity 등)로 음원을 직접 잘라서 준비해야 하므로 번거롭다.

본 기능은 업로드할 음원 파일을 모달 내에서 직접 재생·시각화하고, 파형 위에서 프리뷰 구간을 드래그로 지정한 뒤, 업로드 시 해당 구간만 자동으로 잘라서 프리뷰 파일로 저장한다.

---

## 요구사항

### 프리뷰 구간 설정 모달

곡 추가(Add Song) 모달에서 오디오 파일을 선택하면 **프리뷰 구간 설정 UI**가 모달 내에 인라인으로 표시된다.

#### 파형 타임라인

- 선택한 음원 파일의 **전체 파형(waveform)** 을 **수평(가로) 방향**으로 표시한다.
  - 시간 방향: **좌→우** (왼쪽이 0초, 오른쪽이 곡 끝).
  - 차트 에디터의 세로 파형(PixiJS 렌더링)과 달리, 여기서는 가로 파형을 **Canvas 2D** 로 렌더링한다. 프리뷰 구간 설정은 단순 UI이므로 PixiJS는 불필요하다.
- 파형 높이는 고정(80px), 너비는 모달 콘텐츠 영역의 가용 너비에 맞춘다.
- 파형은 Web Audio API의 `AudioContext.decodeAudioData()`로 `AudioBuffer`를 얻은 뒤, 채널 데이터를 다운샘플링하여 렌더링한다. 스테레오 음원은 좌·우 채널의 평균을 사용한다.
- 파형 색상: `#4488ff`(기본), 선택 구간 내: `#66aaff`(밝게).

#### 구간 선택

- 파형 위에 **선택 구간(selection range)** 을 반투명 오버레이(`rgba(68, 136, 255, 0.2)`)로 표시한다.
- 선택 구간의 **시작점**과 **끝점**에 수직 핸들(드래그 바)을 표시한다.
  - 핸들은 높이 전체를 차지하는 2px 너비의 밝은 세로선(`#ffffff`)이며, 핸들 위에 커서가 올라가면 **수평 리사이즈 커서(↔)** 로 변경된다.
  - 핸들의 히트 영역은 시각적 크기보다 좌우 각 6px 넓게 설정하여 조작성을 확보한다.
- **핸들 드래그**: 시작 핸들 또는 끝 핸들을 좌우로 드래그하여 구간을 조절한다.
- **영역 드래그**: 선택 구간의 내부(핸들이 아닌 영역)를 드래그하면 구간 길이를 유지한 채 전체를 좌우로 이동한다.
- **클릭으로 구간 이동**: 선택 구간 밖의 파형을 클릭하면 클릭 지점을 구간의 중앙으로 하여 구간이 이동한다. 구간 길이는 유지된다.

#### 구간 제약

| 제약 | 값 | 설명 |
|------|----|------|
| 기본 구간 길이 | 15초 | 오디오 파일 선택 시 자동 설정되는 초기 구간 길이 |
| 최소 구간 길이 | 5초 | 핸들 리사이즈 시 이보다 짧아질 수 없다 |
| 최대 구간 길이 | 30초 | 핸들 리사이즈 시 이보다 길어질 수 없다 |
| 구간 시작 하한 | 0초 | 음원 시작 이전으로 이동할 수 없다 |
| 구간 끝 상한 | 음원 길이 | 음원 끝 이후로 이동할 수 없다 |

- **초기 구간 위치**: 음원 전체 길이의 25% 지점을 시작점으로, 거기서 15초 후를 끝점으로 설정한다. 음원이 15초보다 짧으면 전체를 선택한다.

#### 재생 컨트롤

- 파형 아래에 **재생 버튼**(▶ / ⏸)을 제공한다. 클릭하면 **선택 구간만** 재생한다.
- 재생 중 현재 위치를 파형 위에 **재생 커서**(1px 흰색 세로선)로 표시한다.
- 재생이 구간 끝에 도달하면 자동으로 정지한다.
- 구간의 시작·끝 시각을 `MM:SS.s` 형식(소수점 1자리)으로 표시한다. 예: `0:45.2 – 1:00.2 (15.0s)`.

### 프리뷰 미설정 허용

- 프리뷰 구간 설정은 **선택 사항**이다. 사용자가 프리뷰 구간을 설정하지 않아도 곡을 업로드할 수 있다.
- 파형 타임라인 위에 **"프리뷰 사용" 체크박스**를 제공한다. 기본값은 체크(활성).
- 체크 해제 시 구간 선택 UI가 비활성화(회색 처리)되고, 업로드 시 프리뷰 파일을 생성하지 않는다.

### 오디오 트리밍

- 곡 업로드(Add Song) 시 프리뷰가 활성화되어 있으면, 선택 구간에 해당하는 오디오를 **클라이언트 측에서 트리밍**하여 별도의 프리뷰 파일로 업로드한다.
- 트리밍은 `OfflineAudioContext`를 사용하여 수행한다:
  1. 선택 구간의 길이로 `OfflineAudioContext`를 생성한다.
  2. 원본 `AudioBuffer`를 소스로 연결하고, 시작 오프셋을 선택 구간의 시작점으로 설정한다.
  3. `startRendering()`으로 트리밍된 `AudioBuffer`를 얻는다.
  4. 트리밍된 `AudioBuffer`를 WAV로 인코딩하여 `Blob`을 생성한다.
- **페이드 인/아웃**: 트리밍된 구간의 앞 0.5초에 선형 페이드 인, 뒤 0.5초에 선형 페이드 아웃을 적용한다. 구간이 1초 미만이면 페이드를 적용하지 않는다.
- **업로드 포맷**: 트리밍 결과는 **WAV** 형식으로 업로드한다. 런타임 인코딩의 단순성을 위해 WAV를 사용하며, 서버 측 트랜스코딩은 추후 검토한다.
- 업로드 경로: 기존 `songs/{song_id}/preview.wav`.

### 기존 프리뷰 파일 입력 제거

- 현재 곡 추가 모달의 "Preview (ogg/mp3)" 파일 입력 필드를 **제거**한다. 프리뷰는 파형 구간 설정으로만 생성한다.

## 상태 흐름

```
오디오 파일 미선택
  → 파형 영역 비표시

오디오 파일 선택
  → AudioBuffer 디코딩
  → 파형 렌더링
  → 초기 구간 자동 설정 (25% 지점, 15초)
  → 구간 조절 가능

Add Song 클릭
  → 프리뷰 활성화?
    → Yes: OfflineAudioContext로 구간 트리밍 + 페이드 적용
           → WAV 인코딩 → Blob 생성
           → 음원 파일 + 프리뷰 파일 병렬 업로드
    → No:  음원 파일만 업로드
  → DB insert (preview_url 포함/미포함)
```

## 기술 고려사항

### Web Audio API 사용

- `AudioContext.decodeAudioData()`로 로컬 파일을 디코딩하여 `AudioBuffer`를 얻는다. 이 버퍼는 파형 렌더링과 재생 모두에 사용한다.
- 파형 렌더링 시 전체 샘플을 Canvas 픽셀 수에 맞게 다운샘플링한다. 각 픽셀 구간의 min/max 값을 계산하여 막대 형태로 그린다.
- 프리뷰 구간 재생은 `AudioBufferSourceNode`의 `start(when, offset, duration)` 파라미터로 구간 재생을 구현한다.

### WAV 인코딩

- `AudioBuffer`의 채널 데이터를 16-bit PCM WAV로 인코딩한다. 외부 라이브러리 없이 직접 WAV 헤더를 작성한다.
- 스테레오 음원은 스테레오를 유지하여 인코딩한다.

### 성능

- 오디오 파일 디코딩은 비동기(`decodeAudioData`)이므로 UI를 블로킹하지 않는다. 디코딩 중 로딩 인디케이터를 표시한다.
- 파형 렌더링은 Canvas 2D API를 사용하며, 구간 변경 시 전체를 다시 그린다. 파형 데이터(다운샘플링 결과)는 한 번 계산 후 캐싱한다.
