# not4k — 프로젝트 에셋 정의

이 문서는 not4k를 구현하기 위해 필요한 에셋을 카테고리별로 정의하고, 각 에셋의 수급 방안을 제시한다.

---

## 1. 비주얼 에셋 — 게임플레이

### 1.1 노트 그래픽

게임의 핵심 시각 요소. 각 노트 타입이 즉시 구분 가능해야 한다.

| 에셋 | 설명 | 비고 |
|------|------|------|
| 싱글 노트 | 직사각형 기본 노트 스프라이트 | 4레인 × 색상/형태 변형 |
| 더블 노트 | 직사각형. 싱글보다 "무겁고 강조된" 느낌 | 강조 방식 미정: 크기 확대, 이중 테두리, 색상 강조 등 (`note-system.md`) |
| 트릴 노트 | 마름모(◇). 싱글/더블(직사각형)과 형태로 구분 | Good◇ 표기와 시각적 연결 (`note-system.md`) |
| 롱노트 헤드 | 롱노트 시작점 (싱글/더블 변형) | 싱글 헤드, 더블 헤드 각각 필요 |
| 롱노트 바디 | 키 유지 구간의 연속 그래픽 | 레인 너비에 맞는 스트레처블(stretchable) 텍스처 |
| 롱노트 테일 | 롱노트 종료점 | 종결/연결 판정 지점 표시 |
| 트릴 롱노트 | 트릴 구간 내의 롱노트 변형 | 트릴 노트 + 롱노트 바디 결합 |
| 더블 롱노트 | 더블 노트의 롱노트 변형 | 더블 헤드 + 바디 + 테일 |
| 롱노트 실패 헤드 (`FailedNoteContainer`) | 유지 실패한 롱노트의 헤드. 무채색 계열 | 싱글/더블 변형 각각 필요 |
| 롱노트 실패 바디 (`FailedBody`) | 유지 실패한 롱노트의 바디. 무채색 계열 | 기존 `#555555` 회색 규칙 확장. 라인·와이어 모두 무채색 |
| 롱노트 실패 테일 (`FailedTerminalCap`) | 유지 실패한 롱노트의 터미널 캡. 무채색 계열 | `FailedBody`와 동일한 무채색 스타일 |
| 더블 부분 입력 노트 (코어 dim) | 더블 노트에서 1개만 입력 시 한쪽 코어가 꺼진 상태 | 숏·롱 모두 해당. 레인 1,2는 `dimLeft`, 레인 3,4는 `dimRight` (`assets-lab/spec.md`) |

### 1.2 트릴 구간

| 에셋 | 설명 | 비고 |

## 7. 수급 방안

### 7.1 자체 제작

가장 많은 노력이 필요하지만 게임의 아이덴티티를 확보하는 데 필수적인 영역.

| 대상 | 이유 |
|------|------|
| 노트 그래픽 전체 (1.1) | not4k의 노트 타입(싱글/더블/트릴/롱)은 기존 리듬게임과 다른 고유 체계. 기존 에셋을 그대로 쓸 수 없음 |
| 트릴 구간 시각 (1.2) | 트릴 구간 개념 자체가 not4k 고유. 해당하는 기존 에셋이 없음 |
| 판정 이펙트 (1.4) | Good◇ 등 고유 판정 체계에 맞는 커스텀 이펙트 필요 |
| 로고 (2.1) | 게임 아이덴티티의 핵심 |
| 키보드 오버레이 (2.4) | 넘버패드/TKL 이중 레이아웃은 not4k 특유의 UI |
| 손 배치 다이어그램 (2.5) | 약중검엄 배치는 not4k 고유 개념 |
| 튜토리얼용 음원 (5.3) | BPM과 박자가 정확히 제어된 교육용 곡 필요 |

**필요 역량**: 2D 그래픽 디자이너 (게임 UI 경험), 사운드 디자이너 (튜토리얼 곡)

### 7.5 콘텐츠 음원 수급 전략

플레이용 음원은 게임의 콘텐츠 그 자체이므로 별도 전략이 필요하다.

#### 핵심 전제: 비영리 무료 게임 우선 공개

not4k는 **비영리 무료 게임**으로 먼저 공개하여 메커니즘의 반응을 검증한다. 유명 곡이 아니어도 게임 메커니즘 자체가 재미있으면 유저가 곡에 애착을 갖게 되는 구조이며, 실제로 Arcaea와 Phigros가 무명~반무명 작곡가의 오리지널 곡으로 성공한 선례가 있다. 따라서 **곡의 유명도보다 차트 퀄리티**가 우선이다.

비영리 전략의 이점:
- **음원 수급이 용이**: CC-NC 라이선스 곡, BMS 작가의 비영리 허용 곡 등 선택지가 넓어짐
- **법적 리스크 최소화**: 비영리 조건만 충족하면 대부분의 라이선스를 만족
- **메커니즘 검증에 집중**: 유저가 게임 자체의 재미에 대해 피드백할 수 있는 환경
- **상업화 전환 시 점진적 교체**: 반응이 검증되면 음원을 상업 가능한 것으로 순차 교체

#### 음원 확보 방법

| 방법 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **비영리 수록 가능 곡 활용** | CC 라이선스(CC BY, CC BY-NC 등) 또는 작곡가가 비영리 사용을 명시적으로 허가한 곡 | 교섭 불필요, 즉시 사용 가능, 실제 리듬게임 수록 실적이 있는 곡 다수 존재 | 상업 전환 시 교체 필요(CC-NC의 경우) |
| **BMS 작가 직접 교섭** | BMS 작곡가에게 Twitter(X) DM 또는 이메일로 비영리 사용 허가 요청 | 리듬게임에 최적화된 곡, 작가 본인이 저작권 보유(JASRAC 미등록)하여 교섭이 단순, 크레딧 표기만으로 허가하는 경우 다수 | 개별 연락 필요, 응답률 불확실 |
| **CC 라이선스 음원 (해외)** | Newgrounds Audio, SoundCloud, Bandcamp 등에서 CC 라이선스로 공개된 곡 | 글로벌 아티스트 풀, 장르 다양성 확보 | 리듬게임용이 아닐 수 있어 차트 적합성 검토 필요 |
| **자체 작곡** | 팀 내 작곡가 또는 전담 외주 | 게임에 완벽히 맞는 곡, 독점 사용 가능 | 비용과 시간이 가장 많이 소요 |
| **작곡 콘테스트** | Arcaea 모델: 곡 공모전을 열어 입상곡을 수록 | 커뮤니티 참여 유도, 대량 확보 가능 | 게임의 인지도가 일정 수준 이상이어야 유효 |

#### 구체적 음원 소스

| 소스 | 유형 | 비고 |
|------|------|------|
| 비영리 음게 무허가 수록 가능 곡 목록 | CC / 비영리 허용 | Dynamix, Hexa Hysteria 등에 실제 수록된 곡 포함 |
| Newgrounds Audio Portal | CC BY-NC 다수 | Geometry Dash가 활용한 모델. 리듬게임 적합 곡 다수 |
| SoundCloud (CC 필터) | CC BY / CC BY-NC | 아티스트별 라이선스 개별 확인 필수 |
| Bandcamp (CC 태그) | CC BY / CC BY-NC | 고품질 인디 음원, 아티스트 직접 교섭도 용이 |
| BMS 작가 (개별 가이드라인) | 작가별 상이 | 작가의 note.com, 공식 사이트, SNS 프로필에서 이용 규약 확인 |
| 일본 프리 BGM 사이트 | 사이트별 상이 | **주의**: 대부분 "BGM 용도"만 무료이며, 리듬게임처럼 곡이 콘텐츠 주체가 되는 경우 별도 허가 필요 |

#### 단계별 수급 계획

| 단계 | 목표 곡수 | 전략 | 예상 비용 |
|------|----------|------|----------|
| **프로토타입** | 3~5곡 | CC0/CC BY 곡 + 비영리 수록 가능 목록에서 선별 | 0원 |
| **알파** | 5~10곡 | BMS 작가 직접 교섭 (크레딧 표기 제안) + CC 곡 추가 | 0~30만원 |
| **베타 (공개 빌드)** | 10곡 이상 | 검증된 곡 라인업 확정 + 튜토리얼 곡 자체 제작 | 10~100만원 |
| **반응 검증 후** | 필요에 따라 | 상업화 시 CC-NC 곡을 상업 가능 곡으로 교체, 오리지널 곡 커미션, 작곡 콘테스트 검토 | 별도 산정 |

**핵심 원칙**: 튜토리얼 곡(Phase 1~4)은 BPM과 박자가 정확히 제어되어야 하므로 자체 제작 또는 단순 구조의 CC0 곡을 활용한다.

## 8. 에셋 우선순위

구현 단계별로 필요한 에셋을 분류한다. **비영리 무료 게임으로 먼저 공개하여 메커니즘을 검증**하는 전략에 맞춰, 초기 단계에서는 무료/오픈 에셋을 최대한 활용하고 반응 검증 후에 투자를 확대한다.

### Phase A: 프로토타입 (최소 플레이 가능)

게임의 핵심 메커니즘(다중키 바인딩, 트릴, 더블)을 검증하기 위한 최소한의 에셋. **비용 0원 목표.**

| 카테고리 | 필요 에셋 | 수급 방법 |
|----------|----------|-----------|
| 노트 | 싱글/롱/트릴/더블 노트의 플레이스홀더 그래픽 | 단순 도형(사각형, 원)으로 자체 제작 |
| 레인 | 4레인 배경, 판정선 | 단색/그라디언트로 자체 제작 |
| 판정 | 판정 텍스트 (텍스트 렌더링) | 시스템 폰트로 대체 |
| 효과음 | 히트 사운드 1종 | Freesound CC0 |
| 폰트 | 기본 UI 폰트 | Noto Sans CJK |
| 음원 | 테스트용 곡 3~5곡 | CC0/CC BY 음원, 비영리 수록 가능 목록에서 선별 |

### Phase B: 알파 (전체 기능 동작, 비영리 공개 빌드)

모든 게임 시스템이 동작하는 **비영리 무료 공개 버전**. 이 단계에서 유저 반응을 검증한다.

| 카테고리 | 추가 필요 에셋 | 수급 방법 |
|----------|---------------|-----------|
| 노트 | 노트 타입별 구분 가능한 디자인 | 자체 제작 |
| 이펙트 | 히트 이펙트, 판정 이펙트 | 자체 제작 |
| UI | 곡 선택, 결과 화면, 키 바인딩 설정 | 자체 제작 |
| 에디터 | 타임라인 그리드, 마커, 모드 아이콘 | 자체 제작 + 오픈소스 아이콘 |
| 효과음 | 판정별 히트 사운드, UI 사운드 | Freesound + 가공 |
| 음원 | 튜토리얼 곡 + 플레이 곡 5~10곡 | BMS 작가 교섭(크레딧 표기) + CC 음원 |
| 크레딧 | 음원/에셋 저작자 표기 화면 | 자체 제작 (라이선스 조건 충족 필수) |

### Phase C: 베타 (품질 확정, 반응 검증 후)

유저 반응이 검증된 후 품질을 끌어올리는 단계. 상업화 여부에 따라 방향이 갈린다.

| 카테고리 | 추가 필요 에셋 | 수급 방법 |
|----------|---------------|-----------|
| 로고 | 최종 로고 | 자체 제작 또는 커미션 |
| 노트 스킨 | 최종 디자인 노트 그래픽 | 커미션 (리듬게임 스킨 디자이너) |
| 이펙트 | 풀콤보 연출, 결과 화면 애니메이션 | 자체 제작 |
| 튜토리얼 | 손 배치 다이어그램, 안내 UI | 자체 제작 |
| 음원 | 10곡 이상 라인업 확정 | 아티스트 교섭 (상업화 시 CC-NC 곡 교체) |
| 자켓 | 곡별 자켓 이미지 | 일러스트레이터 커미션 |

### Phase D: 상업화 전환 (선택적)

Phase B~C에서 반응이 검증되어 상업화를 결정한 경우에만 진행한다.

| 카테고리 | 필요 작업 | 비고 |
|----------|----------|------|
| 음원 교체 | CC-NC 곡을 CC BY 또는 상업 라이선스 곡으로 교체 | 비영리 전용 곡은 이 시점에서 제거 |
| 오리지널 곡 | 커미션 또는 작곡 콘테스트로 독점 곡 확보 | Arcaea 모델 참고 |
| 유료 콘텐츠 | 추가 곡 팩, 스킨 등 수익화 요소 | 기본 게임은 무료 유지 가능 |
| 법률 검토 | 전체 에셋의 상업 사용 가능 여부 재확인 | 폰트, 아이콘, 효과음 포함 |

## 9. 라이선스 관리 원칙

### 기본 원칙

- 모든 외부 에셋은 **라이선스 출처와 조건을 기록**한다.
- CC BY 에셋은 크레딧 화면에 저작자를 표기한다.
- CC0 에셋도 출처를 기록하되, 표기 의무는 없음을 확인한다.
- 폰트 라이선스는 배포 형태(웹폰트 / 임베디드)에 따라 조건이 달라질 수 있으므로 개별 확인한다.

### 비영리 운영 원칙

- **Phase A~B는 비영리 무료 게임으로 운영**한다. 이 단계에서는 CC-NC(비상업) 라이선스 에셋 사용이 가능하다.
- CC-NC 에셋을 사용하는 경우, 해당 에셋에 **`non-commercial-only` 태그를 부여**하여 상업 전환 시 교체 대상임을 명시한다.
- 비영리 조건으로 BMS 작가에게 허가받은 곡은, **허가 범위(비영리/상업, 기간, 크레딧 조건 등)를 서면(이메일, DM 스크린샷 등)으로 보존**한다.
- 상업화 전환(Phase D) 시, 모든 에셋의 라이선스를 재검토하고 비영리 전용 에셋은 교체한다.

### 리듬게임 특수 주의사항

- 일반 프리 BGM 사이트(魔王魂, MusMus 등)의 곡은 대부분 **"BGM 용도"만 무료**이다. 리듬게임처럼 **곡이 콘텐츠의 주체가 되는 경우 별도 허가가 필요**할 수 있으므로, 이용 규약에서 "음악 게임" 또는 "곡이 주체인 사용"의 허용 여부를 반드시 확인한다.
- BMS 작가의 곡은 대부분 JASRAC 미등록(작가 본인이 저작권 보유)이므로 작가와의 직접 교섭만으로 사용 가능하다. 단, JASRAC에 등록된 곡인지 여부는 반드시 확인한다.
