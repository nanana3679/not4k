# not4k — 차트 편집기

이 문서는 차트 편집기의 요구사항과 차트의 데이터 모델을 정의한다.

---

## 기술 스택

| 영역 | 선택 | 역할 |
|------|------|------|
| UI | React 19 (DOM) | 툴바, 모달, 설정 등 일반 UI |
| 타임라인 렌더링 | PixiJS v8 (WebGL Canvas) | 레인, 그리드, 파형, 노트, 스냅선 등 매 프레임 갱신이 필요한 타임라인 |
| 오디오 | Web Audio API (AudioContext) | 음원 재생/정지, 파형 데이터 추출 |

게임 클라이언트와 동일한 React 19 + PixiJS v8 조합을 사용한다. 상세 근거는 `tech-stack.md` §차트 에디터 기술 스택 참조.

---

## 편집기 요구사항

### 타임라인

에디터의 그리드는 다음 레인으로 구성된다.

| 레인 종류   | 개수 | 용도                          |
| ----------- | ---- | ----------------------------- |
| 노트 레인   | 4    | L1~L4 노트를 배치하는 레인   |
| BPM 레인    | 1    | BPM 마커를 배치하는 보조 레인 |
| 박자 레인   | 1    | 박자 마커를 배치하는 보조 레인 |
| 메시지 레인 | 1    | 메시지를 배치하는 보조 레인. 튜토리얼 등에서 특정 구간에 안내 텍스트를 표시하기 위해 사용한다 |

- **시간 방향**: 타임라인은 **아래에서 위로** 시간이 흐른다. time 0(첫 마디)이 하단, 이후 시간이 위로 올라간다.
- 그리드 배경에 음원 파일의 **파형(waveform)** 을 표시한다.
- 파형 위에 BPM·박자 마커로부터 계산된 **박자 그리드**를 겹쳐 표시한다.
  - 그리드 라인은 마디선과 박자선을 구분할 수 있어야 한다.
  - 오프셋(ms)을 반영하여 그리드와 파형의 정렬을 맞춘다.
  - **마디 번호**: 마디선 옆에 마디 번호를 표시한다.
- 사용자는 파형과 그리드의 정렬을 시각적으로 확인하면서 **오프셋을 조절**할 수 있다.
- **유한 레인 배경**: 레인 배경은 실제 타임라인 구간(첫 마디~끝 마디)만 채운다. 음원이 없으면 기본 16마디, 음원이 있으면 음원 길이 기준으로 마디 수를 계산한다. 타임라인 밖은 검은 배경으로 표시된다.
- **여유 공간(패딩)**: 첫 마디 앞과 끝 마디 뒤에 빈 공간(50px)을 두어 시각적 여유를 제공한다.
- **캔버스 정렬**: 캔버스는 컨테이너 내에서 **가로 중앙 정렬**된다.
- **미니맵**: 캔버스 우측에 타임라인 전체를 축소한 **미니맵**이 표시된다. 마우스 휠 스크롤과 동기화된다. 상세는 아래 "미니맵" 참고.

### 미니맵

캔버스 우측에 표시되는 **타임라인 전체의 축소 미리보기**이다. IDE(VS Code 등)의 미니맵과 동일한 개념으로, 차트의 전체 구조를 한눈에 파악하고 빠르게 탐색할 수 있다.

#### 레이아웃

- **위치**: 캔버스의 우측 가장자리에 고정. 기존 스크롤바를 대체한다.
- **너비**: 60px.
- **높이**: 캔버스 높이 전체를 사용한다.
- **배경**: 반투명 어두운 배경(`rgba(0,0,0,0.5)`)으로 타임라인 콘텐츠와 시각적으로 분리한다.

#### 콘텐츠 렌더링

미니맵은 타임라인 전체(첫 마디~끝 마디)를 세로 방향으로 축소하여 표시한다. 축소 비율은 `캔버스 높이 / 전체 타임라인 높이`로 결정된다.

| 요소 | 렌더링 방식 |
|------|------------|
| 노트 레인 영역 | 4개 레인을 미니맵 너비에 비례 축소. 레인 배경색을 표시하여 레인 구분 |
| 싱글 노트 | 해당 레인 위치에 1px 높이의 수평선. 싱글 노트 색상(`#4488ff`) |
| 더블 노트 | 해당 레인 위치에 1px 높이의 수평선. 더블 노트 색상(`#ffcc00`) |
| 트릴 노트 | 해당 레인 위치에 1px 높이의 수평선. 트릴 노트 색상(`#ffffff`) |
| 롱노트 | 시작~끝 구간을 해당 레인에 채움. 해당 롱노트 색상. 50% 불투명도 |
| 마디선 | 전체 너비를 가로지르는 1px 수평선. 흰색 20% 불투명도 |
| 보조 레인(이벤트) | 미니맵에 표시하지 않음. 노트 레인만 표시한다 |

#### 뷰포트 인디케이터

현재 화면에 보이는 영역을 미니맵 위에 **반투명 사각형**으로 표시한다.

- 색상: `rgba(255, 255, 255, 0.15)` (반투명 흰색 채움)
- 테두리: `rgba(255, 255, 255, 0.5)` 1px 실선
- 위치와 높이는 현재 `scrollY`와 캔버스 높이에 비례한다

#### 조작

| 입력 | 동작 |
|------|------|
| 미니맵 클릭 | 클릭한 위치를 뷰포트 중심으로 이동 |
| 미니맵 드래그 | 드래그하는 동안 실시간으로 뷰포트 이동 |
| 뷰포트 인디케이터 드래그 | 인디케이터를 잡고 드래그하여 뷰포트 이동 |

미니맵 위에서의 클릭/드래그는 타임라인의 노트 편집 동작과 독립적이다. 미니맵 영역의 입력은 편집 모드(Create/Select/Delete)에 전달되지 않는다.

### 줌

- 타임라인의 픽셀 밀도는 `pixelPerSecond = zoom` 으로 결정된다. zoom 값은 BPM과 무관하다.
- BPM이 변하면 시간축(픽셀)은 그대로이고 **그리드 간격이 달라진다**.
- 사용자는 **zoom** 값을 조절하여 표시되는 노트 밀도를 변경할 수 있다.
- `Ctrl` + 마우스 휠로 줌을 조절한다. 브라우저 기본 줌은 에디터 마운트 중 차단된다.

### 재생

- 음원의 **재생/정지** 기능을 제공한다.
- **재생 커서**: 현재 재생 위치를 밝은 초록색 수평선으로 표시한다. 커서 우측에 왼쪽을 향하는 삼각형 핸들이 있으며, 핸들을 드래그하여 재생 위치를 변경(seek)할 수 있다.
- **커서 핸들 드래그**: 타임라인 우측(마디 번호 영역)을 클릭하면 해당 위치로 즉시 이동하며, 드래그하면 실시간으로 재생 위치가 갱신된다.

### 스냅

- **스냅(snap)** 기능으로 그리드 눈금 간격을 조절할 수 있다. 기준 단위는 **박(beat)** 이다 (예: 1/4박, 1/8박, 1/16박, …).
- 노트 배치·이동 시 스냅 설정에 따라 가장 가까운 그리드 눈금에 흡착된다.
- **엔티티 판정(hit test)은 스냅과 무관하게 실제 마우스 위치(raw beat)를 사용한다.** 스냅 1/4 설정에서도 1/16 위치에 배치된 노트를 정확히 hover·클릭·삭제할 수 있다.

### 노트 배치 좌표

노트는 그리드 선을 기준으로 **중앙 정렬**된다. 노트 높이가 `H`이고 그리드 위치가 `P`이면, 노트는 `P − H/2` ~ `P + H/2`에 위치한다.

롱노트의 시작점·끝점도 동일한 규칙을 따른다.

```
시작점 S, 끝점 E, 노트 높이 H 일 때:

  S−H/2 ─┬─ 시작 노트 상단
         │
    S    ─── 시작 그리드 선
         │
  S+H/2 ─┴─ 시작 노트 하단 ─┬─ 바디 시작
                             │  바디 길이 = E − S − H
                             │  (E − S < H 이면 바디 없이 시작·끝 노트가 겹침)
  E−H/2 ─┬─ 끝 노트 상단   ─┘─ 바디 끝
         │
    E    ─── 끝 그리드 선
         │
  E+H/2 ─┴─ 끝 노트 하단

  총 시각적 길이 = (E − S) + H
```

시작점 = 끝점(길이 0)인 롱노트의 시각적 높이는 `H`이다.

**Z-order** (뒤→앞):

| 계층 | 요소 | 비고 |
|------|------|------|
| 1 | 레인 배경 | L1~L4·BPM·박자·메시지 레인의 배경색. 레인 구분을 위해 교대 색상 사용 |
| 2 | 파형 | 음원의 waveform |
| 3 | 마디선 | 마디 경계를 나타내는 굵은 수평선 |
| 4 | 박자선 | 마디 내 박자 경계를 나타내는 수평선. 마디선보다 얇다 |
| 5 | 스냅선 | 스냅 설정에 따른 세분 눈금선. 박자선보다 얇고 연하다 |
| 6 | 트릴 구간 | 트릴 롱노트의 배경 영역 |
| 7 | 롱노트 바디 | |
| 8 | 롱노트 끝점 | |
| 9 | 롱노트 시작점(헤드) | |
| 10 | 일반 노트 | 롱노트 헤드보다 위 — 헤드는 바디로 존재를 인지할 수 있지만 일반 노트는 가려지면 알 수 없다 |
| 11 | **선택된** 롱노트 바디 | |
| 12 | **선택된** 롱노트 끝점 | |
| 13 | **선택된** 롱노트 시작점(헤드) | |
| 14 | **선택된** 일반 노트 | |
| 15 | 고스트 프리뷰 | Create 모드에서 배치 예정 위치를 표시하는 반투명 오버레이. 기존 엔티티 위에서는 고스트 대신 hover 아웃라인이 표시된다 |
| 16 | 마디 번호 | 마디선 우측에 표시되는 마디 번호 라벨 |
| 17 | 재생 커서 | 현재 재생 위치의 수평선 + 드래그 핸들 |

`o-o-` 패턴에서 이전 롱노트의 끝점을 조작하려면 해당 롱노트를 먼저 선택하여 z-index를 올린 뒤 리사이즈 핸들을 사용한다.

### 오토스크롤

- **오토스크롤** 토글을 제공한다.
- 활성화 시 재생 중 현재 재생 위치를 따라 타임라인이 자동으로 스크롤된다.

### 편집 모드

다음 세 가지 모드 중 하나를 선택하여 작업한다.

| 모드   | 단축키 | 설명 |
| ------ | ------ | ---- |
| Create | `C`    | 배치할 엔티티 종류(노트 종류·마커 종류)를 선택한 뒤 타임라인을 클릭하여 배치한다. 마커를 생성하면 직전 마커의 값을 복사한 새 마커가 생성되며, 더블클릭으로 값을 수정하는 모달이 열린다. 커서 위치에 **고스트 프리뷰**(반투명 흰색)가 표시된다 |
| Select | `S`    | 엔티티를 선택하여 이동·삭제한다 |
| Delete | `D`    | 엔티티를 클릭하면 삭제한다. 노트를 먼저 시도하고, 노트가 없으면 트릴 구간 삭제를 시도한다(빈 구간만 삭제 가능). PC에서는 우클릭 삭제로 대체 가능하지만, 모바일 환경을 고려하여 별도 모드로 제공한다 |

> **모달 중 단축키 비활성화**: 마커 편집 모달이나 메타데이터 모달이 열려 있을 때는 모드 전환 단축키(`C`/`S`/`D`)를 포함한 모든 편집기 단축키가 비활성화된다.

> **이동(Move)은 별도 모드가 아닌 Select 모드의 기능으로 통합한다.** 선택된 노트를 드래그 앤 드롭하거나 방향키로 이동한다.

### 단축키

#### 모드 전환

| 키 | 동작 |
|----|------|
| `C` | Create 모드 진입 |
| `S` | Select 모드 진입 |
| `D` | Delete 모드 진입 |

#### Create 모드

| 입력 | 동작 |
|------|------|
| 빈 셀 클릭 | **포인트 엔티티**(싱글/더블/트릴 노트, 마커): 해당 위치에 배치. **구간 엔티티**(롱노트, 트릴 구간): 길이 0인 엔티티를 생성. 기존 엔티티 위에서 클릭하면 생성이 무시된다 |
| 빈 셀 클릭 + 드래그 | 구간 엔티티 선택 시, 드래그 방향으로 끝점이 확장되어 길이가 있는 엔티티를 생성. 드래그 중 **구간 고스트 프리뷰**(반투명 흰색 시작점+바디+끝점)가 표시된다. 포인트 엔티티에는 적용되지 않는다 |
| `C` + 마우스 휠 | `C`를 누르고 있는 상태에서 휠을 굴리면 생성할 엔티티 타입을 변경한다. 엔티티 타입 드롭다운이 열린다 |
| 커서 이동 | **빈 셀 위**: 스냅 위치에 **고스트 노트**(반투명 흰색 사각형)가 표시된다. 보조 레인 위에서는 **고스트 마커**가 표시된다. **기존 엔티티 위**: 해당 엔티티에 **hover 아웃라인**(흰색 테두리)이 표시되고, 고스트 노트는 숨겨진다. hover와 고스트는 상호 배타적으로 동작한다. 레인 밖으로 나가거나 다른 모드로 전환하면 고스트가 사라진다 |

#### Select 모드

| 입력 | 동작 |
|------|------|
| 엔티티 클릭 | 해당 엔티티를 선택 (기존 선택은 해제) |
| `Shift` + 클릭 | 기존 선택을 유지한 채, 해당 엔티티를 선택에 추가 |
| `Alt` + 클릭 | 해당 엔티티를 선택에서 제거 |
| 빈 영역에서 드래그 | 드래그 영역(사각형)이 생성되며, 영역과 겹치는 노트들이 선택된다. 기존 선택은 해제된다 |
| `Shift` + 드래그 | 기존 선택을 유지한 채, 드래그 영역과 겹치는 노트들을 선택에 추가한다 |
| `Alt` + 드래그 | 드래그 영역과 겹치는 노트들을 기존 선택에서 제거한다 |
| 노트 외 영역 클릭 | 선택 해제 및 이동 확정 |
| `Enter` | 선택 해제 및 이동 확정. 배치 제약 조건을 만족하지 않으면 이동하지 않고 원래 위치로 복귀 |
| `↑` / `↓` | 선택된 노트의 이동 예정 위치를 한 틱씩 상/하로 이동 |
| `Shift` + `↑` / `↓` | 선택된 롱노트의 끝점을 한 틱씩 상/하로 이동 (리사이즈). 롱노트가 아닌 노트에는 적용되지 않는다 |
| 선택된 노트 드래그 앤 드롭 | 드래그 중 이동될 위치에 프리뷰를 표시하며, 배치 제약 조건 충족 여부를 시각적으로 표시한다. 드롭 시 제약 조건을 만족하면 이동 확정, 불만족 시 원래 위치로 복귀 |
| `Delete` | 선택된 노트를 삭제 |

#### 모드 무관 (공통)

| 입력 | 동작 |
|------|------|
| 노트 우클릭 | 해당 노트를 즉시 삭제. 노트가 없으면 해당 위치의 트릴 구간 삭제를 시도한다(빈 구간만 삭제 가능) |
| 우클릭 드래그 → 노트 진입 | 우클릭을 누르고 있는 상태에서 노트 위로 드래그하면 해당 노트를 삭제. 연속 삭제에 유용 |
| `Ctrl` + 마우스 휠 | 타임라인의 zoom 값을 조절한다. 휠 업으로 확대, 휠 다운으로 축소. 브라우저 기본 줌은 차단된다 |

#### 공통

### 배치 제약 조건

편집기는 다음 제약 조건을 만족하지 않는 배치를 허용하지 않는다.

1. **동일 위치 중복 금지** — 같은 레인·같은 박자수에 두 개 이상의 노트가 존재할 수 없다.
2. **롱노트 구간 내 겹침 금지** — 롱노트 바디의 시작점과 끝점을 제외한 열린 구간 안에 다른 노트나 롱노트 시작점·끝점이 존재할 수 없다.
3. **트릴 구간 전용** — 트릴 구간 안에는 트릴 노트만 존재할 수 있고, 트릴 노트는 트릴 구간 안에만 존재할 수 있다.
4. **트릴 구간 겹침 금지** — 같은 레인에서 트릴 구간끼리 겹칠 수 없다.
5. **이벤트 겹침 금지** — 같은 종류의 이벤트 마커(BPM, 박자표, 메시지, stop)가 같은 박자수에 중복 존재할 수 없다.
6. **stop 구간 내 노트 금지** — stop 이벤트(§정지 (Stop) 이벤트 참조)가 점유하는 구간 안에 노트를 배치할 수 없다.
7. **길이 0 롱노트 헤드 금지** — 길이 0인 롱노트(시작점 = 끝점)에는 헤드 노트가 존재할 수 없다. 길이가 있는 롱노트(시작점 ≠ 끝점)에만 헤드 노트(싱글/더블/트릴)가 시작점에 배치된다.

### 롱노트 상호작용

롱노트는 시작점·바디·끝점으로 구성되는 구간 엔티티이다. 다른 노트와 달리 **시작점과 끝점이 분리**되어 있으므로, 생성·선택·리사이즈에 별도의 상호작용 설계가 필요하다. 이 절의 규칙은 싱글 롱노트, 더블 롱노트, 트릴 롱노트 모두에 동일하게 적용된다.

#### 위치 겹침 문제

롱노트의 시작점에 헤드 노트(싱글/더블/트릴)가 존재하거나, 끝점 위치에 다른 롱노트의 시작점이 올 수 있다(`o-o-` 패턴). 이처럼 시작점·끝점에 다른 노트가 겹칠 수 있으므로, 리사이즈를 위해 끝점을 드래그하려면 **노트 이동**과 **롱노트 리사이즈** 동작을 구분하는 메커니즘이 필요하다.

#### 생성 (Create 모드)

롱노트 타입(싱글 롱/더블 롱/트릴 롱)을 선택한 상태에서:

1. 타임라인의 원하는 위치를 **클릭 후 드래그**한다.
2. 클릭 지점이 시작점(헤드), 드래그 끝 지점이 끝점이 된다.
3. 드래그 중 현재 끝점 위치에 프리뷰가 표시되며, 스냅 설정에 따라 흡착된다.
4. 마우스를 놓으면 배치 제약 조건을 검증하여 배치하거나, 불만족 시 배치하지 않는다.
5. **길이가 0이 아닌 롱노트**(시작점 ≠ 끝점)를 생성하면 시작점에 헤드 노트(싱글/더블/트릴)가 함께 배치된다. **길이 0인 롱노트**(시작점 = 끝점, 클릭만 하고 드래그하지 않은 경우)에는 헤드 노트가 생성되지 않는다.

모바일에서는 타임라인을 탭한 뒤 **즉시 드래그**하여 동일하게 동작한다.

#### 선택 (Select 모드)

롱노트의 **시작점·바디·끝점** 어디를 클릭하든 **롱노트 전체가 하나의 단위로 선택**된다. 롱노트의 구성 요소(시작점, 바디, 끝점)를 개별 선택할 수 없다.

> **근거**: 롱노트의 시작점과 끝점은 항상 쌍으로 존재하며(데이터 모델 상 상호 참조), 헤드 노트가 있는 경우 헤드 노트도 시작점과 같은 위치에 있어야 한다. 구성 요소를 분리 선택하면 데이터 정합성이 깨지므로, 전체를 하나의 단위로 취급한다.

#### 리사이즈 (Select 모드)

**롱노트가 선택되면 리사이즈 핸들이 표시된다.** 리사이즈 핸들을 드래그하여 롱노트의 시작점 또는 끝점을 변경한다.

##### 리사이즈 핸들

| 위치 | 설명 |
|------|------|
| 시작 핸들 | 헤드 노트와 바디의 **경계 지점**(헤드 하단 가장자리)에 표시 |
| 끝 핸들 | 바디 끝점의 **하단 가장자리**에 표시 |

- 핸들은 바디 가장자리의 **얇은 수평 바(bar)** 형태로, 노트 영역과 시각적으로 구분된다.
- 핸들 위에 커서가 올라가면 **수직 리사이즈 커서(↕)**로 변경된다.
- 핸들은 롱노트가 **선택된 상태에서만** 표시된다.

##### 히트 영역 분리

헤드 노트와 시작 핸들이 인접하므로, 히트 영역을 공간적으로 분리한다:

```
 ┌──────────┐  ← 헤드 노트 히트 영역 (드래그 → 롱노트 전체 이동)
 ├══════════┤  ← 시작 핸들 히트 영역 (드래그 → 시작점 리사이즈)
 │  바 디   │  ← 바디 히트 영역 (드래그 → 롱노트 전체 이동)
 │  바 디   │
 ├══════════┤  ← 끝 핸들 히트 영역 (드래그 → 끝점 리사이즈)
 └──────────┘
```

- 시작 핸들은 **헤드 노트의 하단 가장자리**에 위치하여, 헤드 노트 클릭/드래그와 공간적으로 분리된다.
- 끝 핸들은 **바디 하단 가장자리**에 위치한다.
- 핸들의 히트 영역은 시각적 크기보다 상하 각 4px 넓게 설정하여 조작성을 확보한다.

##### 드래그 동작

| 드래그 대상 | 동작 |
|-------------|------|
| 헤드 노트 영역 | 롱노트 전체 이동 (시작점·끝점 동시 이동) |
| 바디 영역 | 롱노트 전체 이동 |
| 시작 핸들 | 시작점(헤드 위치) 변경 — 리사이즈 |
| 끝 핸들 | 끝점 변경 — 리사이즈 |

- 리사이즈 중 프리뷰가 표시되며, 스냅 설정에 따라 흡착된다.
- 리사이즈 결과가 배치 제약 조건을 위반하면 시각적으로 표시하고, 드롭 시 원래 크기로 복귀한다.

##### 방향키 리사이즈

롱노트가 선택된 상태에서 modifier key와 방향키를 조합하여 리사이즈한다:

| 입력 | 동작 |
|------|------|
| `Shift` + `↑` | 끝점을 한 틱 위로 이동 (롱노트 길이 축소) |
| `Shift` + `↓` | 끝점을 한 틱 아래로 이동 (롱노트 길이 확대) |

`↑`/`↓` 단독은 기존 동작(롱노트 전체 이동)을 유지한다. `Shift`를 조합하면 끝점만 이동하여 리사이즈한다.

> **시작점은 방향키로 리사이즈하지 않는다** — 시작점 리사이즈는 핸들 드래그로만 수행한다. 끝점 리사이즈만 방향키로 지원하여 조작의 복잡도를 제한한다.

##### 리사이즈 제약

| 제약 | 설명 |
|------|------|
| 최소 길이 | 시작점과 끝점이 같은 위치(길이 0)를 허용한다(startBeat ≤ endBeat). 길이 0인 롱노트는 시각적 높이 `H`로 표시된다 |
| 역전 방지 | 시작 핸들을 끝점 이후로, 또는 끝 핸들을 시작점 이전으로 드래그할 수 없다. 해당 경계에서 클램핑된다 |
| 배치 제약 조건 | 리사이즈 결과는 배치 제약 조건을 모두 만족해야 한다 |
| 헤드 타입 보존 | 헤드가 있는 롱노트의 시작점을 리사이즈해도 헤드 노트 타입(싱글/더블/트릴)은 변경되지 않는다 |

#### 모바일 터치 조작

| 입력 | 동작 |
|------|------|
| 롱노트 롱프레스 | 롱노트 전체 선택. 리사이즈 핸들이 확대 표시된다 |
| 핸들 드래그 | 리사이즈. 터치 히트 영역은 최소 44×44px |
| 바디 드래그 | 롱노트 전체 이동 |

모바일에서 리사이즈 핸들은 PC보다 크게 표시되며, 터치 대상의 최소 크기(44×44px)를 보장한다.

### 시각적 피드백

#### 롱노트 끝점
- 끝점은 헤드와 동일한 색상이되, **50% 투명도**로 렌더링되어 시각적으로 구분된다.
- 헤드(시작점)와 바디는 100% 불투명도를 유지한다.

#### 선택된 노트
- 선택된 노트(포인트/롱노트)에는 **밝은 윤곽선**이 표시된다.
- 윤곽선은 노트 영역 **안쪽**(alignment: 0)으로 그려져 인접 노트와 겹치지 않는다.

#### 마커 렌더링
- **BPM 마커**: 보라색 배경. 마커 위에 BPM 값을 11px 크기로 **중앙 정렬** 표시한다.
- **박자 마커**: 빨간색 배경. 분모가 1이면 분자만 표시하고(예: `4`), 분모가 1이 아니면 분수로 표시한다(예: `7/2`). 11px 크기로 중앙 정렬.
- **메시지 마커**: 분홍색 배경에 **연분홍 테두리**(안쪽 정렬)가 항상 표시된다. 텍스트는 마커 영역 내에서 워드랩되며, 영역을 초과하면 말줄임표(`…`)로 잘린다.

#### 토스트 알림
- 배치 제약 위반, 삭제 불가 등 경고 상황에서 화면 하단 중앙에 **토스트 메시지**가 표시된다.
- 토스트는 일정 시간 후 자동으로 사라진다.

### 마커 편집 모달

- 마커(BPM·박자·메시지)를 **더블클릭**하면 편집 모달이 열린다.
- **BPM 마커**: BPM 값 입력 필드.
- **박자 마커**: 분자/분모 입력 필드.
- **메시지**: 텍스트 입력 필드.
- 모달에서 **Save**로 값을 저장하거나 **Delete**로 마커를 삭제할 수 있다.
- **beat 0 보호**: 0박에 위치한 BPM 마커와 박자 마커는 삭제할 수 없다(Delete 버튼 비활성화).

### 메타데이터 편집 모달

- 툴바의 **Meta** 버튼으로 메타데이터 편집 모달을 연다.
- 편집 가능 필드: 곡명, 아티스트명, 난이도명, 난이도 레벨, 오프셋(ms), 오디오 파일, 이미지 파일, 프리뷰 오디오 파일.
- **Load Audio**: 모달 내에서 오디오 파일을 로드할 수 있다. 파일 선택 시 파형이 타임라인에 반영되고 오디오 파일명이 메타데이터에 기록된다.

### 툴바 구성

| 영역 | 요소 | 설명 |
|------|------|------|
| 좌측 | Mode 버튼 (Create/Select/Delete) | 현재 모드 하이라이트 |
| 좌측 | Entity Type 드롭다운 | Create 모드에서만 표시 |
| 중앙 | Snap 버튼 (1/4, 1/8, 1/16) | 스냅 분할 선택 |
| 중앙 | Zoom 표시 | 현재 zoom 값 (px/s) |
| 중앙 | Play/Pause 버튼 | 재생 상태 토글 |
| 우측 | Meta 버튼 | 메타데이터 편집 모달 열기 (오디오 로드 포함) |
| 우측 | Save Chart 버튼 | 차트를 Supabase Storage에 업로드 (공개/비공개 선택) |
| 우측 | Save As 버튼 | 현재 차트를 다른 난이도로 복제 저장 (§다른 이름으로 저장 참조) |
| 우측 | Load Chart 버튼 | 서버에서 차트 불러오기 |

### 트릴 구간 삭제 규칙

- 트릴 구간은 **내부에 노트가 없을 때만** 삭제할 수 있다.
- 트릴 구간 안에 노트가 있는 상태에서 삭제를 시도하면 "Zone contains notes — remove them first" 토스트가 표시된다.
- 우클릭 삭제와 Delete 모드 모두 동일한 규칙을 따른다.

### 롱노트 삭제

- 롱노트를 삭제하면 해당 롱노트에 대응하는 트릴 구간(같은 레인·같은 시작점·끝점)도 함께 삭제된다.

### 저장 및 불러오기

- Supabase Storage에 차트를 **업로드**(저장)할 수 있다. Google OAuth(관리자 전용)로 인증한 뒤 업로드한다.
- 서버에 저장된 차트를 **로드**(불러오기)하여 편집을 이어갈 수 있다.

### 다른 이름으로 저장 (Save As)

기존 차트를 복제하여 **다른 난이도의 새 차트**를 생성하는 기능이다. 한 곡에 여러 난이도(EASY/NORMAL/HARD/EXPERT)의 차트를 제작할 때, 기존 차트를 기반으로 수정하여 다른 난이도의 차트를 만드는 워크플로우를 지원한다.

#### 동작

1. 툴바의 **Save As** 버튼을 클릭하면 **Save As 모달**이 열린다.
2. 모달에서 **대상 난이도명**과 **대상 난이도 레벨**을 입력한다.
   - 난이도명 드롭다운: EASY, NORMAL, HARD, EXPERT 중 선택.
   - 난이도 레벨: 1~15 숫자 입력.
   - 현재 편집 중인 난이도와 동일한 난이도명은 선택할 수 없다(비활성화).
3. **Save** 버튼을 누르면:
   - 현재 차트 데이터를 대상 난이도의 경로(`songs/{songId}/{difficulty}.json`)에 새로 저장한다.
   - Extra 노트 데이터도 함께 복제 저장한다.
   - `charts` 테이블에 새 행을 삽입한다(기존 행이 있으면 upsert).
   - 차트의 메타데이터(`difficultyLabel`, `difficultyLevel`)를 대상 값으로 변경하여 저장한다.
   - 자켓 이미지, 프리뷰 음원, 음원 파일은 곡(`songs/{songId}/`) 단위로 공유하므로 복제하지 않는다.
4. 저장 성공 후 **에디터가 새 난이도 차트로 전환**된다. URL 파라미터(`?songId=...&difficulty=...`)가 새 난이도로 갱신되며, 이후 Save Chart 버튼은 새 난이도에 저장한다.

#### 기존 차트 덮어쓰기 경고

대상 난이도에 이미 차트가 존재하는 경우, "이미 {난이도명} 차트가 존재합니다. 덮어쓰시겠습니까?" 확인 다이얼로그를 표시한다. 사용자가 확인하면 기존 차트를 덮어쓰고, 취소하면 모달로 돌아간다.

#### 검증

- Save As 실행 전에도 기존 Save Chart과 동일한 **배치 제약 조건 검증**을 수행한다. 검증 실패 시 저장하지 않고 ValidationErrorModal을 표시한다.
- HARD 전용 패턴(`.-`, `-=`)이 포함된 차트를 EASY 또는 NORMAL로 저장하려 하면 경고를 표시한다. 저장은 허용하되, 사용자가 해당 패턴을 인지하고 수정할 수 있도록 안내한다.

#### 더티 상태

- Save As 성공 후, 새 난이도 기준으로 savedChartSnapshot이 갱신되어 더티 상태가 초기화된다.
- 원본 난이도의 차트는 영향을 받지 않는다(Save As 전에 저장하지 않은 변경 사항은 원본에 반영되지 않음).

### 차트 공개/비공개

- 차트는 **공개** 또는 **비공개** 상태를 가진다.
- **공개 차트**: 곡 선택 화면에 표시되며, 모든 유저가 플레이할 수 있다.
- **비공개 차트**: 곡 선택 화면에 표시되지 않으며, 에디터에서만 접근할 수 있다. 제작 중인 차트를 서버에 저장하되 유저에게 노출하지 않기 위한 장치이다.
- 에디터에서 차트를 업로드할 때 공개/비공개를 선택할 수 있다.

## 엔티티

차트를 구성하는 배치 가능한 요소. 모든 엔티티는 **위치**를 가진다.

### 위치

| 속성   | 설명                                                                 |
| ------ | -------------------------------------------------------------------- |
| 레인   | L1~L4. 노트에는 필수, 박자 마커와 BPM 마커에는 없음 (레인에 속하지 않는 전역 엔티티) |
| 박자수 | 곡 시작(0박)으로부터의 경과 박자. **분수**로 표기한다                 |

**박자수의 분수 표기**: 정수 박자는 `0`, `1`, `4`처럼 표기하고, 박자 내 세분 위치는 분수로 표기한다. 예를 들어 8비트의 두 번째 틱은 `1/2`, 16비트의 세 번째 틱은 `3/4`. 이를 통해 비트 분할에 독립적인 위치 표현이 가능하다.

---

### 노트

레인 위에 배치되는 입력 대상. 위치의 레인이 필수이다.

| 노트 종류              | 설명                                                                |
| ---------------------- | ------------------------------------------------------------------- |
| 싱글 노트              | 레인에 바인딩된 키 중 하나를 누르는 노트. 롱노트 시작 위치에 배치하면 헤드 역할을 한다 |
| 더블 노트              | 레인에 바인딩된 서로 다른 키 2개를 동시에 누르는 노트. 롱노트 시작 위치에 배치하면 헤드 역할을 한다 |
| 트릴 노트              | 직전 입력과 다른 키를 눌러야 하는 노트. 롱노트 시작 위치에 배치하면 헤드 역할을 한다 |
| 싱글 롱노트 시작  | 싱글 롱노트의 시작점. 대응하는 끝점을 참조한다. 헤드 노트 없이 롱노트만 존재하는 것이 기본 |
| 싱글 롱노트 끝    | 싱글 롱노트의 끝점. 대응하는 시작점을 참조한다         |
| 더블 롱노트 시작  | 더블 롱노트의 시작점. 대응하는 끝점을 참조한다. 헤드 노트 없이 롱노트만 존재하는 것이 기본 |
| 더블 롱노트 끝    | 더블 롱노트의 끝점. 대응하는 시작점을 참조한다         |
| 트릴 롱노트 시작  | 트릴 롱노트의 시작점. 대응하는 끝점을 참조한다. 헤드 노트 없이 롱노트만 존재하는 것이 기본 |
| 트릴 롱노트 끝    | 트릴 롱노트의 끝점. 대응하는 시작점을 참조한다         |

**시작점과 끝점의 상호 참조**: 롱노트의 시작점과 끝점은 반드시 쌍으로 존재하며, 서로를 참조한다. 시작점 없는 끝점, 또는 끝점 없는 시작점은 유효하지 않다.

**헤드 구현**: 롱노트에는 헤드가 내장되어 있지 않다. 롱노트 시작점과 같은 레인·같은 박자수에 일반 노트(싱글/더블/트릴)를 배치하면 해당 노트가 헤드 역할을 한다. 헤드 노트와 롱노트는 별개의 엔티티이며, 판정도 독립적이다.

---

### 트릴 구간

트릴 노트가 등장할 수 있는 시각적 영역. 레인 위에 시작점과 끝점을 가진 구간으로 표시된다. 트릴 구간 자체는 입력을 요구하지 않는다. 트릴 롱노트 바디와는 별개의 엔티티이다.

| 종류           | 설명                                                        |
| -------------- | ----------------------------------------------------------- |
| 트릴 구간 시작 | 트릴 구간의 시작점. 대응하는 구간 끝을 참조한다             |
| 트릴 구간 끝   | 트릴 구간의 끝점. 대응하는 구간 시작을 참조한다             |

**시작과 끝의 상호 참조**: 롱노트와 동일하게, 트릴 구간 시작과 끝은 반드시 쌍으로 존재하며 서로를 참조한다.

---

### 박자 마커

한 마디에 몇 박자가 들어가는지(**beatPerMeasure**)를 정의하는 전역 엔티티. 레인에 속하지 않는다. BPM이 한 박자의 시간 간격을 정의하므로, 박자 마커는 한 마디에 필요한 **박자 수만** 정의한다. beatPerMeasure는 정수 또는 분수로 표기하며, 분수는 `7/2`(=3.5박) 등 특수한 박자를 표현하기 위한 장치이다. 이 값을 통해 타임라인의 **마디선 위치가 결정**된다.

- **마디 0에 반드시 존재**한다. 곡의 시작 시점에서 박자가 정의되어야 한다.
- **마디 인덱스로 정의되며, 항상 해당 마디의 첫 박에 위치**한다.
- 다시 등장하면 **해당 시점 이후로 beatPerMeasure가 변경**된다. 다음 박자 마커가 나올 때까지 유효하다.

예: 마디 0에 beatPerMeasure=`4`, 마디 8에 beatPerMeasure=`3` → 마디 0~7은 4박자 마디, 마디 8 이후는 3박자 마디.

---

### BPM 마커

템포를 정의하는 전역 엔티티. 레인에 속하지 않는다.

- **0박에 반드시 존재**한다. 곡의 시작 시점에서 BPM이 정의되어야 한다.
- 다시 등장하면 **해당 시점 이후로 BPM이 변경**된다. 다음 BPM 마커가 나올 때까지 유효하다.

예: 0박에 `BPM 150`, 64박에 `BPM 180` → 0~63박은 150 BPM, 64박 이후는 180 BPM.

---

### 메시지

특정 구간에 안내 텍스트를 표시하는 구간 엔티티. 레인에 속하지 않는 전역 엔티티이며, 메시지 레인에 배치한다. 튜토리얼 차트에서 노트 메커니즘이나 판정 규칙을 설명하는 데 사용한다.

| 종류       | 설명                                                    |
| ---------- | ------------------------------------------------------- |
| 메시지 시작 | 메시지 구간의 시작점. 대응하는 메시지 끝을 참조한다     |
| 메시지 끝   | 메시지 구간의 끝점. 대응하는 메시지 시작을 참조한다     |

| 속성   | 설명                                                                 |
| ------ | -------------------------------------------------------------------- |
| 텍스트 | 표시할 안내 문자열                                                   |
| 박자수 | 시작·끝 각각의 위치. 다른 구간 엔티티와 동일하게 분수 박자로 표기한다 |

- **시작과 끝의 상호 참조**: 트릴 구간·롱노트와 동일하게, 메시지 시작과 끝은 반드시 쌍으로 존재하며 서로를 참조한다.
- **겹침 금지**: 메시지 구간의 시작점과 끝점을 제외한 열린 구간 안에 다른 메시지의 시작점·끝점이 존재할 수 없다. 단, 메시지 구간의 끝점과 다른 메시지 구간의 시작점은 같은 박자수에 위치할 수 있다 (롱노트의 `o-o-` 패턴과 동일).
- **텍스트 편집**: 메시지 시작 또는 끝을 **더블클릭**하면 텍스트 편집 모달이 열린다. 마커 생성과 동일한 방식이다.
- **노트 배치와 독립**: 메시지는 노트 레인에 영향을 주지 않는다. 메시지 구간 안에 어떤 노트든 자유롭게 배치할 수 있다.

---

### 정지 (Stop) 이벤트

특정 시점에서 노트 스크롤을 일시 정지시키는 이벤트. 시작 박자수와 종료 박자수를 가지는 구간 엔티티이다.

- **stop 구간 동안 노트 스크롤이 멈추며**, 종료 시점에서 다시 재개된다.
- stop 구간 내에는 노트를 배치할 수 없다 (배치 제약 조건 6번).
- 다른 구간 엔티티와 동일하게 시작과 끝이 쌍으로 존재하며 서로를 참조한다.

---

### 통합 EventMarker 모델

코드상 BPM 마커, 박자 마커, 메시지, stop 이벤트는 **EventMarker**라는 단일 유니온 타입으로 통합 구현되어 있다. 각 EventMarker는 `type` 필드(`bpm` | `timeSignature` | `message` | `stop`)로 구분되며, 타입에 따라 고유 속성(BPM 값, 박자 분자/분모, 텍스트, 정지 구간 등)을 가진다. 시작/종료 박자수 등 공통 위치 정보를 공유한다. 보조 레인에서 드래그로 생성한다.
