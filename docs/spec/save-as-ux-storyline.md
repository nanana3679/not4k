# Save As (다른 이름으로 저장) — UX 스토리라인

이 문서는 차트 에디터의 "다른 이름으로 저장" 기능의 사용자 경험 시나리오를 기술한다.

---

## 배경

차트 제작자는 한 곡에 대해 여러 난이도(EASY/NORMAL/HARD/EXPERT)의 차트를 만들어야 한다. 대부분의 경우 상위 난이도 차트를 먼저 완성한 뒤, 그것을 기반으로 노트를 간소화하여 하위 난이도를 파생한다. 이 워크플로우를 지원하기 위해 "다른 이름으로 저장(Save As)" 기능이 필요하다.

---

## 시나리오 1: HARD 차트에서 NORMAL 차트 파생

### 전제

- 제작자는 곡 "Starlight"의 HARD (Lv.9) 차트를 완성하여 저장한 상태이다.
- 이제 같은 곡의 NORMAL (Lv.5) 차트를 만들고 싶다.

### 흐름

1. **에디터에서 HARD 차트를 열고 있는 상태**에서 툴바의 **Save As** 버튼을 클릭한다.

2. **Save As 모달**이 열린다.
   - 상단에 현재 차트 정보가 표시된다: `Starlight — HARD Lv.9`
   - 난이도명 드롭다운에 EASY, NORMAL, EXPERT가 표시된다. HARD는 현재 난이도이므로 비활성화(회색, 선택 불가).
   - 난이도 레벨 입력 필드 (기본값: 현재 레벨과 동일한 9).

3. 제작자가 **NORMAL**을 선택하고, 레벨을 **5**로 변경한다.

4. **Save** 버튼을 클릭한다.

5. NORMAL 차트가 아직 존재하지 않으므로 확인 없이 바로 저장이 진행된다.
   - 차트 JSON이 `songs/{songId}/normal.json`에 업로드된다.
   - `charts` 테이블에 `(songId, normal)` 행이 삽입된다.
   - 메타데이터의 `difficultyLabel`이 "NORMAL", `difficultyLevel`이 5로 변경된다.

6. 저장 완료 후:
   - 토스트: "Chart saved as NORMAL Lv.5"
   - URL이 `?songId=...&difficulty=normal`로 갱신된다.
   - 에디터 제목 영역에 "NORMAL Lv.5"가 표시된다.
   - 이후 Save Chart 버튼은 NORMAL에 저장한다.

7. 제작자는 이제 **NORMAL 차트를 편집**한다. HARD에서 가져온 노트 중 복잡한 패턴을 삭제하고 단순화한다.

8. 작업이 끝나면 **Save Chart** 버튼으로 NORMAL 차트를 저장한다.

> **핵심 가치**: 빈 캔버스에서 시작하지 않고, HARD 차트의 구조(BPM 마커, 박자 마커, 메시지, 기본 노트 배치)를 그대로 물려받아 시작할 수 있다.

## 시나리오 2: 기존 차트 덮어쓰기 경고

### 전제

- 곡 "Aurora"에 EASY (Lv.2)와 HARD (Lv.8) 차트가 이미 존재한다.
- 제작자가 HARD 차트를 열어서 수정한 뒤, 실수로 Save As → EASY를 시도한다.

### 흐름

1. HARD 차트에서 Save As 모달을 열고 EASY를 선택한다.

2. Save 버튼을 누르면, **덮어쓰기 확인 다이얼로그**가 표시된다:
   > "EASY 차트가 이미 존재합니다. 덮어쓰시겠습니까?"
   >
   > [덮어쓰기] [취소]

3. 제작자가 **취소**를 누르면 모달로 돌아가고, 아무 변경도 발생하지 않는다.

4. 제작자가 **덮어쓰기**를 누르면 기존 EASY 차트가 새 내용으로 교체된다.

> **핵심 가치**: 이미 완성된 다른 난이도 차트를 실수로 날리는 것을 방지한다.

---

## 시나리오 3: HARD 전용 패턴 경고

### 전제

- 제작자가 HARD 차트에 헤드 없는 롱노트(`.-` 패턴)를 사용하고 있다.
- 이 차트를 NORMAL로 Save As 하려 한다.

### 흐름

1. Save As 모달에서 NORMAL을 선택하고 Save를 누른다.

2. 차트에 HARD 전용 패턴이 포함되어 있으므로 **경고 다이얼로그**가 표시된다:
   > "이 차트에 HARD 전용 패턴(헤드 없는 롱노트)이 N개 포함되어 있습니다.
   > NORMAL 차트에서는 이 패턴을 사용하지 않는 것이 권장됩니다.
   > 저장 후 해당 패턴을 수정해 주세요."
   >
   > [저장 후 수정하기] [취소]

3. **저장 후 수정하기**를 누르면 저장이 진행되고, 제작자가 새 차트에서 해당 패턴을 직접 수정한다.

> **핵심 가치**: 난이도 설계 규칙 위반을 인지시키되, 워크플로우를 차단하지 않는다. 제작자가 복사 후 수정하는 것이 자연스러운 흐름이므로 저장 자체는 허용한다.

---

## 시나리오 4: 미저장 변경 사항이 있는 상태에서 Save As

### 전제

- 제작자가 HARD 차트를 수정 중이다 (더티 상태).
- 아직 Save Chart을 누르지 않은 채 Save As를 시도한다.

### 흐름

1. Save As 버튼을 클릭한다.

2. Save As 모달이 정상적으로 열린다. **미저장 변경 사항 경고**가 모달 상단에 표시된다:
   > "현재 HARD 차트에 저장되지 않은 변경 사항이 있습니다.
   > Save As는 현재 편집 중인 내용을 새 난이도에 저장합니다.
   > 원본 HARD 차트에는 변경 사항이 반영되지 않습니다."

3. 제작자가 이를 확인하고 Save As를 진행한다.
   - 새 난이도에는 현재 편집 중인 (미저장) 내용이 저장된다.
   - 원본 HARD의 서버 데이터는 마지막으로 저장한 상태 그대로 유지된다.

> **핵심 가치**: 원본 차트가 의도치 않게 변경되는 것을 방지하면서, "복사 후 수정" 워크플로우를 자연스럽게 지원한다.

---

## 시나리오 5: 검증 실패

### 전제

- 제작자가 편집 중인 차트에 배치 제약 위반이 존재한다 (예: 롱노트 구간 내 겹침).

### 흐름

1. Save As 버튼 → 모달에서 NORMAL 선택 → Save 클릭.

2. 기존 Save Chart과 동일한 **배치 제약 조건 검증**이 실행된다.

3. 검증 실패 시 Save As 모달이 닫히고 **ValidationErrorModal**이 표시된다.

4. 제작자가 에러를 확인하고, 문제를 수정한 뒤 다시 Save As를 시도한다.

> **핵심 가치**: 유효하지 않은 차트가 어떤 난이도로든 저장되는 것을 방지한다. Save와 Save As의 검증 기준이 동일하다.

## UI 배치

```
┌───────────────────────────────────────────────────────────────┐
│  [Create] [Select] [Delete]   [Snap ▼] [Zoom] [▶ Play]       │
│                                                               │
│                    ┌─ flex spacer ─┐                          │
│                                                               │
│  [Meta] [Save Chart] [Save As] [Delete Chart]                 │
└───────────────────────────────────────────────────────────────┘
```

- **Save As 버튼**은 Save Chart과 Delete Chart 사이에 위치한다.
- Save Chart과 시각적으로 구분하기 위해 Save As 버튼은 **아웃라인 스타일**(배경 투명, 테두리만 표시)로 렌더링한다.
- Save As 버튼은 `activeSongId`가 없으면 비활성화된다 (곡이 선택되지 않은 상태).
