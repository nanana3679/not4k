# 트릴 구간 경계의 입력 추적 보호

## 문제

트릴 구간 경계에서 노트의 실제 입력 시점과 노트의 원래 위치(noteTime)가 다른 구간에 걸치는 상황이 발생할 수 있다.

### 시나리오 1: 구간 밖 노트를 늦게 쳐서 구간 안에서 처리

```
이전 트릴 구간:  [..... t ]        ← 마지막 노트 (noteTime=1900ms)
새 트릴 구간:              [ t  t  t ]  ← 2000ms 시작
실제 입력:                  ^          ← 2010ms에 입력 (110ms 늦음)
```

이전 구간의 마지막 트릴 노트(1900ms)를 플레이어가 늦게 쳐서 새 구간이 시작된 후(2010ms)에 처리했다. 이 입력이 새 구간의 교대 추적에 기록되면:

- 새 구간의 첫 노트에서 같은 키를 눌렀을 때 Good◇(교대 실패)이 발생한다
- 플레이어 입장에서는 이전 구간의 마지막 노트와 새 구간의 첫 노트는 별개의 동작이다
- 이전 노트를 늦게 친 것에 대한 페널티(Good/Bad 판정)는 이미 받았으므로, 새 구간의 교대 판정까지 영향을 받는 것은 이중 페널티이다

### 시나리오 2: 구간 안 노트를 일찍 쳐서 구간 밖에서 처리

```
트릴 구간:       [ t  t  t ]  ← 2000ms 시작, 첫 노트도 2000ms
실제 입력:     ^               ← 1960ms에 입력 (40ms 빠름)
```

트릴 구간 시작과 동시에 등장하는 첫 노트(noteTime=2000ms)를 일찍(1960ms) 입력했다. 이 노트는 원래 트릴 구간에 속하는 노트이므로, 입력 시점이 구간 밖이더라도 교대 추적이 정상 동작해야 한다.

## 해결 원칙

**교대 추적 기록 여부는 노트의 원래 위치(noteTime)를 기준으로 판단한다.**

- `noteTime >= 현재 활성 트릴 구간의 시작 시간` → 교대 추적에 키를 기록한다 (정상)
- `noteTime < 현재 활성 트릴 구간의 시작 시간` → 교대 추적에 키를 기록하지 않는다 (보호)

판정(Perfect/Great/Good/Bad)과 교대 체크(Good◇ 판정)는 기존 로직대로 정상 수행된다. 변경되는 것은 `trillAlternation.set(lane, keyCode)` 호출 여부뿐이다.

## 근거

1. **인지적 경계 존중**: 플레이어는 트릴 구간 밖의 노트와 구간 안의 트릴 노트를 별개의 동작으로 인식한다. 교대 추적도 이 인지적 경계를 존중해야 한다.
2. **이중 페널티 방지**: 늦은 입력에 대한 타이밍 페널티(Good/Bad)는 이미 적용된다. 추가로 새 구간의 교대 판정까지 오염시키는 것은 과도한 처벌이다.
3. **플레이 일관성**: 같은 키 배치에서 입력 타이밍만 약간 다를 때, 교대 판정 결과가 달라지면 플레이의 일관성이 깨진다. 노트의 원래 위치를 기준으로 하면 타이밍 편차에 관계없이 일관된 교대 판정을 보장한다.

## 구현

`JudgmentEngine`에 `trillZoneCurrentStartMs: Map<Lane, number | null>` 필드를 추가하여 현재 활성 트릴 구간의 시작 시간을 추적한다. `processTrillNoteInput()`에서 노트의 `noteTime`이 현재 활성 구간의 시작 시간 이상인지 확인하고, 아닌 경우 `trillAlternation.set()`을 스킵한다.
